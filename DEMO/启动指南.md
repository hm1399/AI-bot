# AI-Bot 后端启动与 WhatsApp 使用指南

## 环境要求

| 依赖 | 最低版本 | 检查命令 |
|------|---------|---------|
| Python | 3.11+ | `python3 --version` |
| Node.js | 20.0+ | `node --version` |
| npm | 随 Node 安装 | `npm --version` |

> 当前机器 Python 版本为 3.9.6，需要升级到 3.11+。推荐用 `brew install python@3.11` 或 pyenv 安装。

---

## 一、首次安装依赖（只需一次）

### 1. Python 后端依赖

```bash
cd server
pip3 install -r requirements.txt
```

### 2. WhatsApp Bridge 依赖

```bash
cd server/bridge
npm install
npm run build
```

---

## 二、配置 API Key

后端需要 LLM API Key 才能运行 AI 对话。当前配置使用 OpenRouter。

API Key 已保存在 `server/.env` 文件中，无需额外配置。如果需要更换：

```bash
# 编辑 server/.env
OPENROUTER_API_KEY=你的key
```

---

## 三、启动步骤（每次使用都要做）

需要打开 **两个终端窗口**，按顺序启动。

### 终端 1：启动 WhatsApp Bridge

```bash
cd server/bridge
npm start
```

**首次运行**会在终端显示一个二维码：

```
🐈 nanobot WhatsApp Bridge
========================

🌉 Bridge server listening on ws://127.0.0.1:3001

📱 Scan this QR code with WhatsApp (Linked Devices):

▄▄▄▄▄▄▄▄▄▄▄▄▄
█ ▀▀▀ █ ... █
...
```

**扫码步骤**：
1. 打开手机 WhatsApp
2. 点击右上角 **⋮** → **Linked Devices**（已关联的设备）
3. 点击 **Link a Device**（关联设备）
4. 扫描终端中的二维码

扫码成功后终端显示：
```
✅ Connected to WhatsApp
```

> 扫码信息保存在 `server/bridge/whatsapp-auth/`，下次启动不用重新扫码。
> 如果连接失败，删除该目录重新扫码：`rm -rf server/bridge/whatsapp-auth`

### 终端 2：启动 Python 后端

```bash
cd server
python3 main.py
```

启动成功后显示：
```
AI-Bot 服务端 v0.6.0
服务已启动: http://0.0.0.0:8765
  模型: z-ai/glm-4.7 (openrouter)
  ASR: medium | TTS: zh-CN-XiaoxiaoNeural
  健康检查: http://localhost:8765/api/health
  WebSocket: ws://localhost:8765/ws/device
WhatsApp channel 已启用，bridge: ws://localhost:3001
```

## 四、使用 WhatsApp AI 助手

当前配置为 **self_only 模式**——AI 只在你自己和自己的聊天中响应，别人给你发消息不会触发 AI。

### 打开 Self Chat（给自己发消息）

1. 打开 WhatsApp
2. 找到顶部的 **"Message yourself"**（给自己发消息）聊天
   - 如果找不到，点右下角新建聊天，搜索自己的名字
3. 在这个聊天中发送任何消息，AI 就会回复

### 示例对话

```
你：今天天气怎么样？
AI：你好！我目前无法直接查询天气，需要用搜索工具帮你查。你在哪个城市？

你：帮我打开浏览器
AI：好的，正在打开浏览器。

你：写一段Python代码，计算斐波那契数列
AI：（返回代码）
```

## 五、关闭服务

1. 在终端 2（Python 后端）按 `Ctrl+C`
2. 在终端 1（WhatsApp Bridge）按 `Ctrl+C`

## 六、常见问题

### Bridge 启动后没有显示二维码
- 确保在**交互式终端**中运行（不要在后台运行）
- 如果之前扫过码，不会再显示二维码，直接显示 `✅ Connected`

### Python 后端报错 "WhatsApp bridge connection error"
- 确保终端 1 的 Bridge 已经启动并显示 `Bridge server listening`
- 检查端口 3001 没有被占用：`lsof -i :3001`

### AI 不回复消息
- 确认你是在 **"Message yourself"** 聊天中发送的
- 检查终端 2 的日志，看是否有 `self_only mode: ignoring` 的提示
- 检查 API Key 是否有效

### 需要重新扫码
```bash
rm -rf server/bridge/whatsapp-auth
# 然后重新启动 Bridge
```

### 端口被占用
```bash
# 查看占用端口的进程
lsof -i :8765    # Python 后端
lsof -i :3001    # WhatsApp Bridge

# 结束进程
kill <PID>
```

## 七、配置文件速查

| 文件 | 用途 |
|------|------|
| `server/config.yaml` | 主配置（模型、端口、WhatsApp开关） |
| `server/.env` | API Key |
| `server/workspace/SOUL.md` | AI 人格设定（名字叫"小博"） |
| `server/bridge/whatsapp-auth/` | WhatsApp 登录凭证（自动生成） |

### 切换 AI 模型

编辑 `server/config.yaml`：

```yaml
nanobot:
  model: z-ai/glm-4.7        # 当前：智谱GLM（通过OpenRouter）
  # model: anthropic/claude-sonnet-4-5  # 替换为 Claude
  # model: openai/gpt-4o          # 替换为 GPT-4o
  provider: openrouter
```

### 切换 WhatsApp 模式

编辑 `server/config.yaml`：

```yaml
whatsapp:
  self_only: true       # true = 只响应自己  |  false = 按 allow_from 控制
  allow_from: ["*"]     # self_only=false 时生效："*" 允许所有人
```

```
├── main.py                  # Python 后端入口
├── config.py                # 配置加载
├── config.yaml              # 主配置文件
├── .env                     # API Key
├── requirements.txt         # Python 依赖
├── bridge/                  # WhatsApp Bridge (Node.js)
│   ├── package.json
│   ├── tsconfig.json
│   ├── src/                 # TypeScript 源码
│   │   ├── index.ts         # Bridge 入口
│   │   ├── server.ts        # WebSocket 服务端
│   │   └── whatsapp.ts      # Baileys WhatsApp 客户端
│   └── dist/                # 编译输出（npm run build 生成）
├── nanobot/                 # AI 框架核心
│   ├── channels/
│   │   ├── whatsapp.py      # WhatsApp Python 端
│   │   └── base.py          # 频道基类
│   ├── agent/               # AI Agent 循环
│   ├── config/              # 配置 schema
│   └── ...
├── channels/
│   └── device_channel.py    # ESP32 硬件通信
├── services/
│   ├── asr.py               # 语音识别 (Whisper)
│   └── tts.py               # 语音合成 (Edge-TTS)
└── workspace/
    ├── SOUL.md              # AI 人格设定
    ├── sessions/            # 对话历史
    └── config.json          # 自动生成的 nanobot 配置
```
