## 一、架构总览

**MVP 阶段为两端系统：硬件设备（ESP32-S3） + 电脑服务端（Python）。**

手机 App（Flutter）为后期扩展，MVP 不涉及。

核心思路：**以 nanobot 的 AgentLoop + 工具系统作为 AI 引擎**，在其上层添加 DeviceChannel（ESP32 WebSocket 通道）+ ASR/TTS 服务。nanobot 源码复制到项目中直接修改，删除不需要的 Channel（Telegram/Discord 等），保留电脑控制和工具系统核心。

```
┌──────────────┐    WiFi局域网 (WebSocket)    ┌───────────────────────────────────┐
│  ESP32-S3    │◄────────────────────────────►│   电脑服务端 (Python)              │
│  硬件设备     │  二进制音频帧 + JSON控制指令    │   aiohttp 单端口 :8765             │
│              │  TTS音频流 + 屏幕/LED控制      │                                   │
└──────────────┘                              └───────────────────────────────────┘
                                                │ 内部模块
                                                ├── DeviceChannel (WebSocket 端点)
                                                ├── faster-whisper (ASR 语音识别，本地)
                                                ├── Edge-TTS (语音合成，云端免费)
                                                ├── nanobot AgentLoop (AI 对话引擎)
                                                │   ├── LiteLLM Provider → Claude API
                                                │   ├── exec 工具 (电脑控制)
                                                │   ├── 文件工具 (read/write/edit/list_dir)
                                                │   ├── 网络工具 (web_search/web_fetch)
                                                │   └── cron 工具 (定时任务)
                                                ├── SessionManager (JSONL 对话持久化)
                                                └── MemoryStore (MEMORY.md 长期记忆)
```

---

## 二、关键架构决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| AI 引擎 | nanobot AgentLoop（复制源码到项目中修改） | 已有完整的 tool_use 循环、会话管理、记忆系统，不需要从零搭 |
| LLM 调用 | 保留 nanobot LiteLLM Provider | 支持多家 LLM，已封装好，省事 |
| 消息架构 | 保留 MessageBus + 自定义 DeviceChannel | 解耦 Channel 和 AgentLoop，干净 |
| 会话/记忆 | nanobot 内置（JSONL + MEMORY.md + HISTORY.md） | 已有 LLM 摘要压缩，不需要 SQLite conversations 表 |
| Web 框架 | aiohttp（单端口 8765） | nanobot 已依赖 aiohttp，不需要引入 FastAPI |
| ASR | faster-whisper 本地 | 免费、离线、无需 API Key |
| TTS | Edge-TTS | 免费、语音质量好（需网络） |
| 电脑控制 | nanobot 内置 exec 工具 | 自带安全防护（拒绝 rm -rf、shutdown 等） |
| 自定义工具 | MVP 不做（task_queue/events 后期加） | 先跑通语音对话 + 电脑控制 |
| MVP 范围 | 只做硬件端，App 后期 | 先专注核心功能 |

## 三、服务端架构

### 3.1 数据流：一次完整的语音交互

```
ESP32 麦克风录音（触摸 → 录音 → 触摸结束）
    │
    │ 二进制 PCM 音频帧 (WebSocket)
    ▼
DeviceChannel._handle_ws()
    │ 累积音频帧到 buffer
    │ 收到 {"type": "audio_end"} → 触发识别
    ▼
asr.transcribe(wav_bytes)         ← faster-whisper 线程池
    │ "帮我打开微信"
    ▼
DeviceChannel._handle_message()
    │ 构造 InboundMessage → MessageBus.inbound
    ▼
AgentLoop.run()
    │ consume_inbound() → _process_message()
    │ ContextBuilder 组装 system prompt + 对话历史
    │ LLM Provider → Claude API (with tools)
    │
    │ Claude 返回 tool_use: exec("open -a WeChat")
    │ → ExecTool 执行 shell 命令
    │ → 结果反馈给 Claude
    │ → Claude 生成最终回复: "好的，微信已经打开了"
    │
    │ OutboundMessage → MessageBus.outbound
    ▼
DeviceChannel.send(msg)
    │ ① 发 display_update → ESP32 屏幕显示文字
    │ ② 发 state_change → ESP32 切换到 SPEAKING 状态
    │ ③ tts.synthesize(text) → Edge-TTS 生成音频
    │ ④ 发 audio_play + 二进制音频帧 → ESP32 播放
    │ ⑤ 发 audio_play_end + state_change IDLE
    ▼
ESP32 扬声器播放语音回复
```

**精简原则：**
- 删除大部分内置 Channel（telegram, discord, slack, feishu, dingtalk, email, qq, matrix, mochat）
- 保留 WhatsApp Channel（用于调试：手机 WhatsApp 发消息测试 AI 对话）
- 删除 ChannelManager（只有一个 DeviceChannel，main.py 直接管理）
- 删除 CLI 命令模块（nanobot CLI 不需要）
- 删除 heartbeat 服务
- 删除 templates 模块
- 保留：agent/、bus/、providers/、session/、config/、cron/、utils/

### 3.3 WebSocket 端点（单端口 :8765）

| 路径 | 用途 |
|------|------|
| `/ws/device` | ESP32 硬件 WebSocket 连接 |
| `/api/health` | 服务在线检测（供未来 App 服务发现用） |

### 3.4 WebSocket 消息协议

**设备 → 服务端：**

| type | data | 传输方式 | 说明 |
|------|------|----------|------|
| (二进制帧) | PCM 音频数据 | Binary | 录音中持续发送 |
| `audio_end` | `{}` | JSON | 录音结束，触发 ASR |
| `touch_event` | `{ gesture: "single/double/long" }` | JSON | 触摸事件 |
| `shake_event` | `{}` | JSON | 摇一摇 |
| `device_status` | `{ battery: 85, rssi: -45 }` | JSON | 状态上报 |

**服务端 → 设备：**

| type | data | 传输方式 | 说明 |
|------|------|----------|------|
| `state_change` | `{ state: "IDLE/PROCESSING/SPEAKING" }` | JSON | 状态切换 |
| `display_update` | `{ text: "AI回复内容" }` | JSON | 屏幕显示 |
| `led_control` | `{ mode: "idle/thinking/speaking" }` | JSON | LED 灯效 |
| `audio_play` | `{}` | JSON | 音频播放开始标志 |
| (二进制帧) | WAV 音频数据 | Binary | TTS 音频流（1KB 分块） |
| `audio_play_end` | `{}` | JSON | 音频播放结束 |

## 四、nanobot 核心机制（保留部分）

### 4.1 AgentLoop 处理流程

```
收到 InboundMessage
    ↓
加载/创建 Session（基于 session_key）
    ↓
ContextBuilder 组装 LLM 请求：
  system_prompt = 核心身份 + SOUL.md + MEMORY.md + Skills
  messages = 对话历史（未压缩部分） + 当前用户消息
    ↓
while iteration < max_iterations:
    调用 LLM Provider（Claude API + tools 定义）
    ↓
    if Claude 返回 tool_use:
        执行工具 → 结果加入 messages → 继续循环
    else:
        拿到最终回复 → 跳出循环
    ↓
保存本轮对话到 Session
触发记忆压缩（如果未压缩消息数 >= memory_window）
    ↓
发布 OutboundMessage → MessageBus → DeviceChannel
```

### 4.2 工具系统

| 工具 | 来源 | 功能 | 安全限制 |
|------|------|------|----------|
| `exec` | nanobot 内置 | 执行 shell 命令 | 拒绝 rm -rf、shutdown、格式化等危险命令；超时 60s |
| `read_file` | nanobot 内置 | 读文件内容 | 可限制 workspace |
| `write_file` | nanobot 内置 | 写文件 | 可限制 workspace |
| `edit_file` | nanobot 内置 | 编辑文件（查找替换） | 可限制 workspace |
| `list_dir` | nanobot 内置 | 列出目录内容 | 可限制 workspace |
| `web_search` | nanobot 内置 | 搜索网页（Brave API） | 需要 Brave API Key |
| `web_fetch` | nanobot 内置 | 获取网页内容 | — |
| `cron` | nanobot 内置 | 创建/列出/删除定时任务 | — |
| `message` | nanobot 内置 | 发送消息到 Channel | — |
| `spawn` | nanobot 内置 | 创建后台子 Agent | — |

### 4.3 会话与记忆

**Session（短期）：**
- JSONL 文件存储，每个 session_key 一个文件
- append-only，不修改历史消息（LLM 缓存友好）
- `last_consolidated` 指针标记压缩进度

**Memory（长期）：**
- `MEMORY.md`：LLM 生成的长期记忆摘要，每次对话都注入 system prompt
- `HISTORY.md`：带时间戳的对话日志，可 grep 搜索
- 触发条件：未压缩消息数 >= memory_window（默认 100）
- 压缩过程：LLM 分析旧消息 → 更新 MEMORY.md + 追加 HISTORY.md

## 六、AI 人格（SOUL.md）

```markdown
# AI-Bot 桌面助手

你是一个桌面 AI 助手，名字叫小博，运行在用户电脑上。
用户通过语音（ESP32硬件设备）与你交流。

## 你的特点
- 回复精简：语音场景下每次回复 1-2 句话，不超过 50 字
- 语言：默认中文，用户用英文则用英文回复
- 性格：主动、友好、实用，不废话

## 你的能力
- 控制电脑：打开关闭应用、文件操作、系统命令（通过 exec 工具）
- 读写文件（通过 read_file / write_file 工具）
- 搜索网页（通过 web_search 工具）
- 定时任务（通过 cron 工具）

## 注意
- 执行 exec 命令前，如果有破坏性（删除文件等），先向用户确认
- 不执行 rm -rf、shutdown 等危险命令
```

## 七、技术栈汇总

| 层 | 技术 | 用途 |
|----|------|------|
| 硬件固件 | Arduino / ESP-IDF (C++) | ESP32-S3 固件 |
| 服务端入口 | Python + aiohttp | WebSocket 服务（单端口 8765） |
| AI 引擎 | nanobot AgentLoop（精简版） | 对话 + tool_use 循环 |
| LLM 调用 | LiteLLM → Claude API | AI 对话 |
| 语音识别 | faster-whisper (本地, base 模型) | 音频转文字 |
| 语音合成 | Edge-TTS (微软免费) | 文字转语音 |
| 电脑控制 | nanobot exec 工具 | shell 命令执行 |
| 会话存储 | JSONL 文件 | 对话历史持久化 |
| 长期记忆 | MEMORY.md + HISTORY.md | LLM 摘要压缩 |
