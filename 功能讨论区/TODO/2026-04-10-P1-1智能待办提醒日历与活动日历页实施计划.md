# P1-1 智能待办提醒日历与活动日历页 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成 `P1-1 智能待办 / 提醒 / 日历` 仍未收口的产品能力，并在前端左侧导航新增一个“活动和日历”页面，提供可视化月历与按日行程列表。

**Architecture:** 本轮继续沿用现有 `Flutter App + aiohttp AppRuntimeService + AppResourceService + PlanningProjectionService + ReminderScheduler` 主干，不推翻现有资源模型，也不破坏当前 Linear 风格 UI。后端重点收口 reminder 调度语义、bundle/source/linked resource 编排与 planning 查询；前端重点新增 `Agenda` 页面、抽出统一 planning 编辑器、移除 workbench 的伪完整 fallback 口径。

**Tech Stack:** Flutter + Riverpod + GoRouter，Python `aiohttp`，本地 JSON store，`PlanningProjectionService`，`ReminderScheduler`，Nanobot planning tool。

## 调研结论摘要

### 前端现状

- 现有导航由 `app/lib/config/routes.dart` + `app/lib/widgets/common/app_scaffold.dart` + `app/lib/widgets/common/app_sidebar.dart` + `app/lib/widgets/common/app_bottom_dock.dart` 统一管理，新增页面必须同时接入路由、桌面 sidebar 和移动端 dock。
- 现有 planning 数据流已经存在：`app/lib/services/planning/planning_service.dart` 通过 `planning/overview`、`planning/timeline`、`planning/conflicts` 拉取后端 projection，`app/lib/providers/app_providers.dart` 负责加载和 WebSocket 事件驱动刷新。
- `app/lib/screens/tasks/tasks_screen.dart` 当前虽然展示 planning workbench，但仍保留本地 fallback timeline/conflict 拼装，且提醒编辑仍写死在 `app/lib/screens/control_center/control_center_screen.dart`。
- `TaskModel / EventModel / ReminderModel` 已支持 `bundle_id / created_via / source_* / linked_*` 字段，说明前端模型层无需重建，重点在编辑流和页面承载方式。

### 后端现状

- `server/services/app_runtime.py` 已提供 `planning/overview`、`planning/timeline`、`planning/conflicts` 接口，并在资源变更或 reminder 触发后刷新 planning state。
- `server/services/planning/planning_bundle_service.py`、`planning_projection_service.py`、`planning_summary_service.py` 已有骨架。
- `server/services/reminder_scheduler.py` 已支持 `sync / trigger / snooze / complete / repeat` 的核心机制，但 `overdue` 规则尚未真正产品化。
- `server/nanobot/agent/tools/planning.py` 和 `server/bootstrap.py` 已接入 planning tool，但当前 agent 创建 Task/Event/Reminder 时并没有把 `bundle_id / source / linked resource` 真正写回资源，文档口径与实际代码不完全一致。

### 与旧计划的冲突说明

- 旧计划 `功能讨论区/TODO/2026-04-08-P1智能待办提醒日历实施计划-macos-app.md` 明确写了“不新增 `/app/planning` 或 `/app/calendar` 路由”。
- 这次按你的新要求执行，新增独立页面，计划采用新增 `/app/agenda` 路由和导航入口，而不是沿用旧约束。
