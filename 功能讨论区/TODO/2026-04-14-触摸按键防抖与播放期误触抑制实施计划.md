# 触摸按键防抖与播放期误触抑制实施计划

> 仅覆盖本轮新增规则：修复设备触摸按键在按压与设备说话期间出现的抖动/误触问题，不改现有前后端协议与 UI。

## 目标

- [x] 触摸按键输入增加稳定采样防抖，避免按下/释放边缘抖动。
- [x] 设备播放语音期间抑制触摸误触发，避免屏幕状态在说话时来回跳。
- [x] 单击、长按说话、长按配对、摇一摇等既有交互保持可用。
- [x] 不修改前端和后端协议，只在固件输入判定层收口。

## 调研结论

- 当前 `handleTouch()` 直接用 `touchRead(TOUCH_PIN) > TOUCH_THRESHOLD` 作为即时真值，没有任何 debounce 或 hysteresis。
- 当前 `canSendVoiceTouch()` / `canSendTapEvent()` 没有避开 `playbackActive`，设备自己说话时产生的触摸噪声也可能被当成有效输入。
- 因此本轮最小可行方案应同时覆盖：
  - 触摸稳定采样防抖
  - 播放期误触抑制

## 推荐方案

- [x] 在 `demo.ino` 为触摸输入新增“原始读数状态”和“稳定状态”分离。
- [x] 只有原始读数连续稳定一段时间后，才更新为真正的 `touchPressed`。
- [x] 设备播放语音期间，禁止新触摸动作进入 `tap / long_press` 判定。
- [x] 保留已有 `TOUCH_HOLD_TRIGGER_MS / REPAIR_TOUCH_HOLD_MS / TOUCH_TAP_WINDOW_MS` 的时序语义。

## 实施范围

- `firmware/arduino/demo/demo.ino`

## 落地步骤

- [x] 新增触摸防抖所需状态变量与时间常量。
- [x] 将 `handleTouch()` 改为“原始状态变化后等待稳定窗口，再更新按压状态”。
- [x] 在可发送触摸事件的判定函数中加入 `playbackActive` 抑制。
- [x] 确保长按配对、长按说话、tap 序列仍沿用稳定后的按键状态。

## 验收口径

- [ ] 手按触摸按键时，屏幕状态不应快速来回抖动。
- [ ] 设备播报语音期间，不应被喇叭震动误触发成新的 touch 事件。
- [ ] 长按触摸仍能正常触发说话。
- [ ] 长按 5 秒仍能进入配对。
- [ ] 摇一摇与字幕滚动逻辑不受影响。
