# 设备屏幕字幕恢复为仅 AI 回复实施计划

> 仅覆盖本轮用户明确要求：撤回前端补加字幕，恢复设备屏幕底部字幕，并限制为只显示 AI 回复正文。

## 目标

- [x] 撤回 `Control Center` 中本轮新增的 `Device Commands` 底部字幕补丁。
- [x] 恢复设备屏幕底部字幕绘制能力。
- [x] 设备底部字幕仅在收到 `text_reply` 时更新。
- [x] 设备语音回复链路也会补发 `text_reply`，避免“会说话但没字幕”。
- [x] `display_update`、配对提示、连接提示、录音/处理中提示不再写入设备底部字幕。
- [x] 字幕保留最后一句，直到下一次 AI 回复覆盖。

## 实施范围

- 前端：
  - `app/lib/screens/control_center/control_center_screen.dart`
- 固件：
  - `firmware/arduino/demo/face_display.h`
  - `firmware/arduino/demo/face_display.cpp`
  - `firmware/arduino/demo/demo.ino`
- 后端：
  - `server/channels/device_channel.py`

## 落地步骤

- [x] 回退前端 `Device Commands` 卡片底部字幕相关新增代码，保留其他本轮有效改动。
- [x] 在 `face_display` 内补回底部字幕区域绘制与缓存。
- [x] 调整 `demo.ino` 的消息分流：
  - `text_reply` 更新底部字幕；
  - `display_update` 只做状态流转，不改底部字幕；
  - 其他本地状态文案不再调用底部字幕更新。
- [x] 调整 `device_channel` 的设备语音回复路径：
  - TTS 语音回复也先补发 `text_reply`；
  - 固件仍然只认 `text_reply`，不回退到 `display_update` 做字幕源。
- [x] 复查改动范围，确认没有把状态提示重新混进字幕来源。

## 验收口径

- [ ] 前端 `Control Center` 不再显示本轮临时补加的设备字幕区。
- [ ] 设备收到 AI 正式回复时，底部出现该句字幕。
- [ ] 设备进入连接中、配对中、录音中、处理中等状态时，底部字幕不被覆盖。
- [ ] AI 说完后，底部保留最后一句，直到下一次回复覆盖。
