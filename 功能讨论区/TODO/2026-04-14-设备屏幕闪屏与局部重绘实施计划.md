# 设备屏幕闪屏与局部重绘实施计划

> 仅覆盖本轮新增问题：设备眨眼、说话嘴型、processing 点点、字幕滚动时出现整屏闪烁，需要把显示更新从整屏重绘改为局部重绘。

## 目标

- [x] 解决设备在动画帧切换和字幕滚动时的整屏闪屏。
- [x] 保持现有眨眼、说话、processing 点点动画行为不变。
- [x] 保持底部 `status > AI reply` 优先级和字幕滚动行为不变。
- [x] 不改前后端协议，只在固件显示层收口。

## 调研结论

- 当前闪屏根因在 `firmware/arduino/demo/face_display.cpp`：
  - `faceUpdate()` 只要动画或字幕有变化，就会触发 `drawCurrentFace()`。
  - `drawCurrentFace()` 每次都会先 `fillScreen(theme.background)`，再整屏重画。
- 因此每一帧嘴型变化、processing 点点推进、字幕偏移推进，都会发生一次“整屏擦除 + 整屏重绘”，这是当前闪屏的直接来源。

## 推荐方案

- [x] 保留“状态切换时整屏重绘”的路径。
- [x] 将“动画帧推进 / 字幕变化”改为局部重绘：
  - 眼睛区域单独重绘
  - 嘴巴区域单独重绘
  - processing 点点区域单独重绘
  - 底部字幕区域单独重绘
- [x] 底部字幕区域无论是否有文字，都先重画/清除字幕区域，避免残影。

## 实施范围

- `firmware/arduino/demo/face_display.cpp`

## 落地步骤

- [x] 拆出整屏重绘函数和局部区域重绘函数。
- [x] 在 `faceUpdate()` 中按具体变化类型触发对应局部重绘，而不是统一整屏重绘。
- [x] 在字幕写入接口中改为只重绘字幕区域，不再整屏刷新。
- [x] 保持 `faceSetState()` 仍执行整屏重绘，确保状态切换时脸部整体一致。

## 验收口径

- [ ] 眨眼时屏幕不应整屏闪一下。
- [ ] 说话嘴型开合时屏幕不应整屏闪一下。
- [ ] processing 点点动画推进时屏幕不应整屏闪一下。
- [ ] 长字幕滚动时屏幕不应整屏闪一下。
- [ ] 状态切换、字幕优先级、字幕滚动仍正常。
