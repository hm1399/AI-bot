# P0 剩余硬件协同与设备控制闭环 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 只完成当前总 TODO 里尚未真正收口的 P0 剩余闭环：`时间/天气/设备状态稳定刷新` 与 `前端发起的设备控制命令真实执行并可回读`。

**Architecture:** 保持现有 `Flutter App -> app-v1 runtime/device API -> DeviceChannel -> demo 固件` 主链路，不新增第二套控制面。后端补设备命令结果协议与运行态读回，固件补命令消费与主动状态上报，前端只在现有 Linear 风格 UI 上把“accepted”升级为“执行结果 + 真实状态”。

**Tech Stack:** Flutter + Riverpod, Python aiohttp backend, ESP32-S3 Arduino demo firmware, WebSocket JSON protocol

## 调研结论

- `server/channels/device_channel.py` 已能发送 `device_command`，但 `get_snapshot()` 只有 `connected/state/battery/wifi_rssi/charging/reconnect_count`，没有 `volume / mute / led / sleep / status_bar` 真回读，也没有命令执行结果协议。
- `server/services/app_runtime.py` 的 `/api/app/v1/device/commands` 当前只返回并广播 `device.command.accepted`，前端无法区分“已受理”和“已执行”。
- `app/lib/providers/app_providers.dart` 对设备命令事件只更新 `globalMessage` 为 accepted 文案；`app/lib/screens/control_center/control_center_screen.dart` 目前还是用 settings 草稿值去驱动 UI。
- `firmware/arduino/demo/demo.ino` 目前会处理 `text_reply / state_change / display_update / status_bar_update / audio_play`，但不会消费 `device_command`，也没有主动发送 `device_status` 心跳。
- 服务端已有 `_time_push_loop()` 和 `_weather_push_loop()`，但“稳定刷新”还差三件事：设备主动状态上报、天气状态可观测、前端/运行态能看见最后一次刷新结果。

## File Structure

- `server/models/protocol.py`
  负责设备 <-> 服务端 WebSocket 消息类型定义；本轮要扩设备命令结果消息。
- `server/channels/device_channel.py`
  负责设备连接、状态缓存、状态栏推送、命令下发；本轮要补命令结果回收、状态栏/控制状态快照、设备状态心跳接入。
- `server/services/app_runtime.py`
  负责 app-v1 runtime 和事件广播；本轮要把设备控制结果与稳定刷新状态暴露给 App。
- `server/tests/test_app_runtime.py`
  负责 app runtime / device route 回归；本轮要补命令回执与运行态快照测试。
- `firmware/arduino/demo/demo.ino`
  负责 demo 固件 WebSocket 收发、屏幕/喇叭/触摸；本轮要补 `device_command` 消费、命令执行结果上报和 `device_status` 主动心跳。
- `app/lib/models/home/runtime_state_model.dart`
  负责解析 runtime 里的设备快照；本轮要补设备控制状态、状态栏信息、命令结果字段。
- `app/lib/providers/app_providers.dart`
  负责事件流整合和设备控制动作；本轮要把 accepted-only 升级为 pending/succeeded/failed。
- `app/lib/screens/control_center/control_center_screen.dart`
  负责设备控制 UI；本轮要改成“显示真实读回状态 + 命令执行结果”，不再把 settings 草稿当成已生效状态。
- `app/lib/widgets/home/device_card.dart`
  负责 Home 的设备概览；本轮要补最后状态栏时间/天气和控制状态读回。
- `功能讨论区/2026-04-06-总TODO清单-功能实现优化与硬件协同.md`
  后续实现完成后回填 P0-5 勾选状态。
- `功能讨论区/2026-04-06-P0语音闭环验收记录-按住设备说话+电脑麦克风代采.md`
  后续实现完成后补“状态刷新 + 设备控制闭环”的联调记录。

```text

```

