# 设备底部英文状态提示与 AI 回复优先级实施计划

> 仅覆盖本轮新增规则：设备底部同时支持英文 status 提示与 AI 回复字幕，显示优先级为 `status > AI reply`，且配对完成后恢复显示上一句 AI 回复。

## 目标

- [x] 设备底部新增双缓冲显示能力：`statusText` 与 `replyText`。
- [x] 底部显示优先级固定为 `statusText > replyText`。
- [x] `statusText` 持续显示，直到下一个 status 替换或被明确清空。
- [x] 配对完成后清空 `statusText`，恢复显示上一句 AI 回复。
- [x] `status` 统一使用英文口径，不再显示中文状态提示。
- [x] 保留现有 AI 回复字幕链路与眨眼/说话/processing 动画。

## 实施范围

- 固件显示层：
  - `firmware/arduino/demo/face_display.h`
  - `firmware/arduino/demo/face_display.cpp`
- 固件状态映射层：
  - `firmware/arduino/demo/demo.ino`
- 后端设备语音回传兼容：
  - `server/channels/device_channel.py`

## 落地步骤

- [x] 在 `face_display` 内把单字幕缓冲改成双缓冲：
  - `reply subtitle`
  - `status subtitle`
- [x] 渲染逻辑改成：有 `status subtitle` 就显示它，否则显示 `reply subtitle`。
- [x] 增加清空 `status subtitle` 的接口，供配对完成时回退到上一句 AI 回复。
- [x] 在 `demo.ino` 中把状态提示写入 `status subtitle`，把 `text_reply` 写入 `reply subtitle`。
- [x] 在 `demo.ino` 中把现有状态提示统一映射成英文，包括至少：
  - `Please connect to computer`
  - `Hold touch button to pair`
  - `Connecting to WiFi`
  - `WiFi connected`
  - `Processing`
- [x] 在合适的“配对完成/进入正常运行态”时机清空 `status subtitle`，恢复显示上一句 AI 回复。
- [x] 保持 `device_channel.py` 的 TTS 回复补发 `text_reply`，避免设备会说话但没有 reply 字幕。

## 验收口径

- [ ] 设备未完成配对时，底部可显示英文状态提示。
- [ ] 设备处理配对/WiFi/处理中时，底部优先显示英文 status，而不是旧的 AI reply。
- [ ] 配对完成并进入正常运行态后，底部不再停留在配对 status，而是恢复显示上一句 AI 回复。
- [ ] 新的 AI 回复到来时，reply buffer 正常更新。
- [ ] 眨眼/说话/processing 动画仍正常，字幕不会被重绘丢失。
