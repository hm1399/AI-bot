# 设备断服后长按重配恢复实施计划

> 仅覆盖本轮新增规则：修复设备在已配对但服务器断开时无法通过长按 5 秒进入配对模式的问题，不修改前端 UI、不修改后端配对协议。

## 目标

- [x] 已配对设备在 `Server disconnected` / 断服场景下，仍可通过长按 5 秒进入配对模式。
- [x] 已连接服务器时，240ms 长按说话逻辑保持原样，不误触发重配。
- [x] 仅修改固件触摸状态机，不扩大到前端和后端行为变更。

## 调研结论

- 当前固件在读取到已配对配置后，会进入 `PAIRING_DISABLED` 正常运行态。
- WebSocket 断开时，屏幕只会提示 `服务器已断开`，但 `pairingPhase` 不会回到 `PAIRING_IDLE`。
- `canArmPairingFromTouch()` 当前只允许“未配对”或“已经在 `PAIRING_IDLE`”时通过 5 秒长按进入配对，因此已配对但断服的设备会被卡死在不可恢复状态。

## 推荐方案

- [x] 保持 240ms 长按说话仍要求 `wsConnected == true`。
- [x] 放开 5 秒重配入口：当设备已配对、当前处于 `PAIRING_DISABLED`，且未建立 WebSocket 时，也允许长按进入配对。
- [x] 继续复用现有 `enterPairingMode()` / `stopRuntimeForPairing()`，避免新增配对状态或协议分支。
