# Agenda 时间本地化与跨天冲突可见性修复计划

> 仅覆盖本轮新增问题：个人行程里语音创建的 `下午 3 点考试` 被显示成 `7:00 AM - 9:00 AM`，且后端提示有冲突时，前端按天视图没有把实际冲突中的跨天事件显示出来，造成用户可见行为偏差。要求保持现有 Agenda / Tasks UI 风格，不擅自改视觉样式，只修时间与可见性口径。

## 目标

- [x] 修复 Agenda / Tasks 中 planning 时间的本地化解析，避免 `+08:00` 时间被当成 UTC 直接显示成早上时段。
- [x] 修复按天日程对跨天事件的过滤逻辑，让覆盖当天的事件在当天视图可见。
- [x] 让前端可见的日程与后端冲突判定口径一致，减少“明明没冲突却提示冲突”的感知偏差。

## 调研结论

- [x] 后端日志显示语音创建 `考试` 时，实际落库时间是 `2026-04-18T15:00:00` 到 `2026-04-18T17:00:00`，语音识别与 planning tool 归一化本身没有把 `下午 3 点` 识别错。
- [x] 后端 planning timeline 投影会把该事件输出为 `2026-04-18T15:00:00+08:00` 到 `2026-04-18T17:00:00+08:00`，说明后端时间口径是本地时区正确值。
- [x] Flutter 侧多个 planning / event / task / reminder 模型的 `_tryParseDateTime()` 仅做 `DateTime.tryParse(value)`，没有在 `parsed.isUtc` 时转 `toLocal()`；Dart 对带 offset 的 ISO 字符串会转成 UTC，直接格式化会显示成 `7:00 AM - 9:00 AM`。
- [x] 当前 Agenda 数据集 `entriesForDay()` 只按 `scheduledAt` 与目标日是否同一天来过滤，跨天事件只会在开始当天显示，不会在覆盖的中间日期显示。
- [x] 本次冲突提示实际命中了已有事件 `event_28c53ef5`：`2026-04-16T20:52:00.000` 到 `2026-04-30T20:52:00.000`；后端冲突判定并非乱报，但前端 `4/18` 的当天视图把这个跨天事件隐藏了，所以用户看不到冲突来源。

## 修复方案

- [x] 前端时间解析：
  - 为 planning / events / tasks / reminders 的时间解析统一补上“UTC 转本地”的兜底。
  - 保持现有文案和时间格式，不改 chip 样式与布局。
- [x] 前端 Agenda 可见性：
  - 把按天过滤从“只看开始时间同日”改成“事件窗口覆盖当天即可显示”。
  - 任务和提醒继续按原有单点时间逻辑过滤，避免扩大范围误伤。
- [x] 回归测试：
  - 补充前端测试，覆盖带 `+08:00` 的时间在 UI 中显示为本地下午时段。
  - 补充前端测试，覆盖跨天事件会出现在覆盖日期当天的数据集中。
  - 按当前用户约束，本轮先补测试，不执行，待你确认修改后再跑。
