# 2026-04-08 WhatsApp 改为可选辅助通道实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 `python3 main.py` 的硬件主链路默认不再依赖 WhatsApp bridge，WhatsApp 只作为可选的辅助消息通道。

**Architecture:** 保持现有 WhatsApp 通道实现不删，但把默认配置和工作流口径改成“默认关闭、按需启用”。主链路只依赖 `main.py + 设备`，当用户显式启用 `whatsapp.enabled=true` 时，再启动 bridge 并使用镜像转发能力。

**Tech Stack:** Python, aiohttp, YAML 配置, 现有 WhatsAppChannel / UnifiedOutboundRouter, Markdown 文档
