# Chat 页面体验与状态区折叠收口实施计划

> 仅覆盖本轮新增需求：把 Chat 页顶部的 `Conversation Experience` 与聊天区顶部状态显示统一收进可折叠区域，默认收起；同时让聊天记录区域尽量占用更多高度，不改现有 UI 风格，不改后端行为。

## 目标

- [x] `Conversation Experience` 改为可收缩卡片，默认收起。
- [x] 当前聊天区顶部的状态显示也并入同一个可收缩区域。
- [x] 收起后聊天记录区域获得更多可视高度，保留现有 header、消息列表、输入框样式语言。

## 调研结论

- `ChatScreen` 当前结构是 `Header -> ExperienceChipBar -> ConversationPanel`，顶部固定占高会直接压缩消息列表可视区域。
- `Conversation Experience` 由 `app/lib/widgets/chat/experience_chip_bar.dart` 独立渲染，适合原地扩展为“头部始终可见 + 内容可折叠”的组件。
- 状态显示目前在 `app/lib/widgets/chat/chat_status_strip.dart` 中，并嵌入 `ConversationPanel` 顶部；若要一起收起，需要把它移动到折叠区容器中，由 Chat 页面统一控制展示。
- 消息列表本身已经在 `Expanded` 中，主要空间损耗来自顶部两个固定块和它们之间的垂直间距。

## 推荐方案

- [x] 新增一个前端本地 UI 折叠状态，默认 `collapsed=true`，仅作用于 Chat 页当前会话界面，不做持久化。
- [x] 将 `ExperienceChipBar` 改造为可折叠容器：始终显示标题、状态 pills 和展开/收起按钮；展开时再显示说明文案、`Scene`、`Persona` 与状态 strip 内容。
- [x] 把 `ChatStatusStrip` 从 `ConversationPanel` 顶部移入折叠容器，保持现有 pill 与 voice activity 展示逻辑，但默认跟随折叠状态隐藏。
- [x] 压缩折叠区与消息面板之间的固定空白，确保 `ConversationPanel` 继续吃掉剩余高度，让消息列表成为主区域。
