# 2026-04-08 main 单进程内置桌面麦克风实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 `server/main.py` 所在服务端进程直接接管本机麦克风采集，以后用户只运行 `python3 main.py` 就能完成“设备触发电脑麦克风代采”。

**Architecture:** 保留现有 `DesktopVoiceService` 作为语音状态机和设备触发桥，但把 `desktop_voice_client.py` 里的本机 `sounddevice` 采音能力内嵌到服务端运行时。设备触发 `long_press / long_release` 后，不再依赖独立 `/ws/desktop-voice` 客户端连接，而是在服务端内部直接开始/结束本机录音并沿用现有 ASR/Agent/TTS 流程。

**Tech Stack:** Python, aiohttp, asyncio, sounddevice, 现有 `DesktopVoiceService`

## 风险点

- `sounddevice` 在服务端启动时拿不到系统麦克风权限，会导致硬件触发后无法录音
- 如果同时保留旧 websocket client 并启用本机内置采音，需要明确优先级，避免双重录音
- 如果直接删掉 `/ws/desktop-voice`，会破坏已有调试方式；本次应优先保留兼容
