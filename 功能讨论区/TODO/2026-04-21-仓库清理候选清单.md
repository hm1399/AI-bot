# 2026-04-21 仓库清理候选清单

## 说明

- 本清单只用于辅助决定“先清哪里”，不代表本轮已经执行清理。
- 本清单按“优先清理”和“更适合归档 / 标识”区分，方便后续分批处理。
- 本轮基于当前仓库实际内容调研，不跑测试，不修改业务代码。

## 优先清理

- `venv/`
  - 本地虚拟环境，包含安装包和字节码，不应作为仓库源码保留。
- `.manager/`
  - 仅包含 `server.log`、`server.pid` 等运行态文件，属于本地状态。
- `.pytest_cache/`
  - pytest 生成缓存，不属于源码。
- `app/.dart_tool/`
  - Flutter 生成缓存目录。
- `app/build/`
  - Flutter 构建产物目录。
- `app/.flutter-plugins-dependencies`
  - Flutter 生成的依赖快照文件。
- `app/macos/Pods/`
  - CocoaPods 本地依赖缓存；若无刻意 vendoring 需求，不适合长期保留。
- `server/workspace/runtime/`
  - 运行时 JSON 状态数据，不是源码。
- `server/workspace/state.sqlite3`
  - 运行时数据库，连同 `state.sqlite3-wal`、`state.sqlite3-shm` 都属于运行产物。
- `server/workspace/config.json`
  - 由 `config.yaml` 生成，不是主配置源。
- `server/bridge/node_modules/`
  - 可重装依赖缓存。
- `APP for BASS 44.zip`
  - 与目录版内容重复，保留一个即可。

## 更适合归档 / 重命名 / 明确标识

- `APP for BASS 44/`
  - 旧前端原型，不是当前主运行路径。
- `App Builder (Copy)/`
  - 老的 App Builder / Figma 代码包，名称和内容都偏历史参考物。
- `nanobot-src/`
  - 上游参考快照，且是嵌套 Git 仓库，更适合归档到 reference / archive 语义下。
- `presentation/`
  - 展示材料，不是运行源码，更适合与课程资料归档到一起。
- `Project_proposal/`
  - 课程 proposal 材料，适合归档。
- `DEMO/`
  - 有价值，但更像演示资料区，而不是主运行目录。
