# 设备屏幕背光亮但内容不显示排查计划

> **For agentic workers:** 本任务按系统化排查执行，先收集证据，再给出单点修复；涉及并行调研时，subagents 使用 `gpt-5.4` + `xhigh`。

**Goal:** 定位并修复“ESP32-S3 设备背光亮起但 ST7789 屏幕没有任何内容显示”的根因，确认是固件渲染链路、`TFT_eSPI` 配置、烧录参数，还是硬件连接/串口占用导致的假象。

**Scope:** 只覆盖本次黑屏排查，不顺手改配对流程、聊天逻辑、控制中心或其它 UI。

---

## 现象

- 设备已成功烧录最新 `firmware/arduino/demo/demo.ino`
- 屏幕背光有亮
- 但屏幕没有文字、状态栏或其它内容显示

## 调研路径

- [ ] 记录复现口径，收集串口启动日志与屏幕初始化证据
- [ ] 对比 `demo.ino`、`face_display.*`、`TFT_eSPI` 配置与硬件文档
- [ ] 排查最近修改和本次 `arduino-cli` 烧录参数是否影响显示链路
- [ ] 确认单一根因并实施最小修复
- [ ] 回填进度并等待用户确认后再执行 git

## subagent 分工

- `Worker A`：只读分析 `firmware/arduino/demo/**` 显示初始化与首次渲染调用链
