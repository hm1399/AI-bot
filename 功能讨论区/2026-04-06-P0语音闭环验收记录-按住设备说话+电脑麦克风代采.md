# 2026-04-06 P0 语音闭环验收记录：按住设备说话 + 电脑麦克风代采

## 目标

- 只按当前主线验收：`按住设备说话` 触发、`电脑麦克风` 负责实际采音、`设备` 负责文字与状态反馈。
- 不默认硬件麦克风已经恢复，不把 `设备本地麦克风直采` 当作本轮前提。
- 不在本文猜测最终实现细节；正式接入方式、输出方式、联调结论由主线程按实际落地结果回填。

## 角色口径

- 设备：主交互入口，负责 `按住` 触发和 `listening / transcribing / responding` 状态反馈。
- 电脑麦克风：当前真实采音源。
- 服务端：接收 PCM、执行 ASR、Agent、TTS/文字回复分发。
- 输出方式：由主线程回填实际采用方案。
  - 方案 A：电脑本地扬声器播放 TTS，同时设备显示回复与状态。
  - 方案 B：设备负责文字与状态反馈，语音暂不播。

## 验收前置条件

- [ ] 服务端已启动，且当前语音入口已由主线程确认。
- [ ] 设备在线，可看到状态变化或文字反馈。
- [ ] 电脑麦克风桥接端已启动，或已经具备手动开始/停止录音能力。
- [ ] 当前输出方式已确认并记录为 `A` 或 `B`。
- [ ] 本次联调使用的分支/提交、设备固件版本、桌面语音客户端版本已准备好记录。

## 主线验收步骤

| 步骤 | 操作 | 预期反馈 | 结果记录 |
| --- | --- | --- | --- |
| 1 | 确认设备在线并处于待机页 | 设备可见在线/待命状态；桌面语音桥接端处于待命 | 记录开始时间、设备状态、桥接端状态 |
| 2 | 用户按住设备说话入口 | 设备立即进入 `listening` 或等价收音态 | 记录设备状态字段、屏幕文案、开始录音时间 |
| 3 | 用户朝电脑麦克风说一句完整的话 | 桌面语音桥接端开始采音，并保持 `16kHz / 16bit / mono PCM` 上送 | 记录录音开始/结束时间、采音来源、是否有 PCM 上传证据 |
| 4 | 用户松开设备或触发停止录音 | 设备退出收音态，进入 `transcribing` 或等价识别处理中状态 | 记录停止录音动作、状态切换耗时 |
| 5 | 检查服务端链路 | 服务端成功收到 PCM，ASR 产出文字 | 记录日志位置、ASR 文本、是否有失败或重试 |
| 6 | 检查回复生成阶段 | Agent 返回回复；设备进入 `responding` 或等价反馈态 | 记录回复文本、返回耗时、状态文案 |
| 7 | 检查最终用户可感知结果 | 至少一种形式完成结果反馈：电脑扬声器播报或设备文字/屏幕提示 | 记录实际输出方式、是否成功显示/播报 |
| 8 | 结束本轮并留档 | 本轮有明确通过/失败结论，并记录问题与证据路径 | 记录 PASS / FAIL / BLOCKED、问题描述、证据路径 |

## 最低通过标准

- [ ] `按住设备说话` 能稳定触发本轮语音入口。
- [ ] 设备能进入收音态，且用户能感知到正在听。
- [ ] 电脑麦克风采集到真实音频，并有 PCM 上传证据。
- [ ] ASR 能把本轮语音转成文字。
- [ ] 设备能进入识别/处理中反馈态。
- [ ] Agent 能返回回复。
- [ ] 设备至少能显示回复，或完成一种可感知的播报/提示。
- [ ] 本轮结果已记录回 `功能讨论区/`，不再沿用“硬件麦克风已可用”的旧假设。
