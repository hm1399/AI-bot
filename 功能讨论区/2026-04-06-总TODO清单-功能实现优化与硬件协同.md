## 当前进度快照

### 前端当前进度

已完成：

- Flutter 工程主骨架已成立，已具备 `Connect / Home / Chat / Tasks / Control Center / Settings / Demo Mode` 页面。
- 已接入 `bootstrap / runtime / sessions / settings / tasks / events / notifications / reminders / device / app events websocket` 主协议。
- 已有本地连接保存、事件流重连、`last_event_id` 恢复、Demo Mode、本地 provider 状态管理。
- app 已完成 Linear 风格重构，补齐深浅主题切换、桌面侧边栏 / 移动端 dock 响应式壳层，以及首页运行态总览和快速操作。
- Chat 页面已补齐会话创建、切换、当前会话可视化入口，并可直接展示结构化 planning 结果。
- `/app/tasks` 已从原来的 task/event 列表升级为 planning workbench，统一展示今日概览、任务、时间线、提醒与冲突。
- `/app/control` 已补齐音量、灯光亮度/颜色、唤醒、休眠、静音等设备命令入口；Notifications / Reminders 已支持已读、删除与启停。
- 本机 Flutter 工具链已恢复可用，`flutter pub get`、`flutter analyze`、`flutter test test/widget_test.dart` 已在 2026-04-09 跑通。

当前缺口：

- `app/lib/services/chat/voice_capture_service.dart` 仍是 stub；当前已经落地的是“硬件按住触发 -> 服务端内置电脑麦克风代采”，但 Flutter App / 手机麦克风自己的真实采集仍未实现。
- 连接页明确保留了 `mDNS / zeroconf` 为后续事项。
- planning workbench 的手动创建 / 编辑入口还需要继续打磨，尤其是和 AI 创建项共享 `bundle_id`、联动 Task / Reminder / Event 的 UI 行为。
- 多会话产品化仍未收口，当前已支持创建和切换，但自动标题、pin / archive 还没补齐。
- 自动化验证仍然偏弱，当前前端虽然已恢复 analyze 和基础 widget test，但 provider / service / websocket / 页面交互测试还缺失。

### 后端当前进度

已完成：

- `server/services/app_runtime.py` 已提供完整 `app-v1` HTTP + WebSocket 接口主干。
- Sessions、Messages、Settings、Tasks、Events、Notifications、Reminders、Runtime、Device、Capabilities 已接入。
- App 鉴权、事件流 replay、聊天入队与回流、设备命令入口、设备状态快照已具备。
- `server/services/app_api/resource_service.py` 已有本地 JSON 持久化 CRUD。
- `server/services/desktop_voice_service.py` 已接入独立 `/ws/desktop-voice` 入口，并支持由 `main.py` 服务端进程内置接管本机桌面麦克风。
- planning 派生层、planning API 和 Agent planning tool 已落地，后端可提供 overview / timeline / conflicts，并支持 `create_task`、`create_event`、`create_reminder`、`complete_task`、`snooze_reminder`、`list_today`。
- reminder 已接入真实调度，触发后会同步更新 reminder 状态、notification、planning 与 summary，不再只是 JSON CRUD。

当前缺口：

- 电脑麦克风实时采集的正式接入链路已经落地，但当前主要是“硬件触发 / 服务端内置采音”与兼容调试入口，App 自己的实时语音采集仍未接上。
- `server/tools/test_client.py` 只能发送音频文件或模拟事件，不是实时电脑麦克风采集工具。
- `wake_word`、`auto_listen` 目前主要停留在设置字段，没有真正检测或状态机闭环。
- reminder 的 repeat / overdue 行为契约还没有完全收口，当前调度主线已通，但产品规则还需继续明确。
- 人格系统、场景模式、外部日历同步、手机推送通知都还没有产品级实现。
- `server/services/app_runtime.py` 继续膨胀，后续维护成本会越来越高。
