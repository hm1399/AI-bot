# Demo 搭建步骤（一步一步）

> 目标：ESP32 设备说话 → 电脑 ASR 识别 → AI 回复 → WhatsApp 显示文字 + 屏幕显示
>
> 前提：ESP32 硬件已焊接完成，串口/WiFi/麦克风/触摸/屏幕已通过测试

---

## 第一步：确认电脑 IP 地址

ESP32 和电脑必须在同一个局域网。先查电脑 IP，后面要填到固件里。

**操作：**

打开终端，运行：

```bash
ipconfig getifaddr en0
```

记下输出的 IP 地址（类似 `192.168.1.xxx`），后面要用。

如果没有输出，试试 `ipconfig getifaddr en1`，或者去 **系统设置 → Wi-Fi → 详细信息** 查看 IP。

---

## 第二步：电脑连接指定 WiFi

ESP32 固件里写的 WiFi 是 `EE3070_P1615_1`（密码 `EE3070P1615`）。

**你的电脑也必须连这个 WiFi**，否则两者不在同一个局域网，无法通信。

连好后重新运行第一步的命令确认 IP。

---

## 第三步：安装 Arduino 库

打开 Arduino IDE，安装以下 3 个库（如果之前测试时已装则跳过）：

1. **TFT_eSPI** — 屏幕驱动（测试2已装过）
2. **WebSockets** by Markus Sattler — 搜 `WebSockets`，作者是 Markus Sattler（不是 ArduinoWebsockets）
3. **ArduinoJson** by Benoit Blanchon — 搜 `ArduinoJson`

**安装方法：** Arduino IDE → 工具 → 管理库 → 搜索库名 → 安装

### TFT_eSPI 配置

如果测试2屏幕已经能正常显示，说明 `User_Setup.h` 已配置好，跳过此步。

如果没有配置过，需要修改 TFT_eSPI 库的 `User_Setup.h`：

```
找到 TFT_eSPI 库文件夹（Arduino IDE → 文件 → 首选项 → 项目文件夹位置 → 打开 libraries/TFT_eSPI/）
编辑 User_Setup.h：
```

```cpp
#define ST7789_DRIVER
#define TFT_WIDTH  240
#define TFT_HEIGHT 240
#define TFT_MOSI 11
#define TFT_SCLK 12
#define TFT_CS   9
#define TFT_DC   10
#define TFT_RST  46
#define SPI_FREQUENCY 40000000
#define TFT_INVERSION_ON
```

## 第四步：修改固件 IP 并烧录

1. 用 Arduino IDE 打开 `firmware/arduino/demo/demo.ino`

2. 找到第 24 行：
   ```cpp
   const char* WS_HOST = "192.168.1.100";  // ← 改为你电脑的局域网 IP
   ```
   把 `192.168.1.100` 改成第一步查到的电脑 IP

3. 确认 Arduino IDE 设置：
   - 开发板：`ESP32S3 Dev Module`
   - USB CDC On Boot：`Enabled`
   - Flash Size：`16MB`
   - PSRAM：`OPI PSRAM`
   - Upload Speed：`921600`
   - 端口：选你的 ESP32 串口

4. 点 **上传**，等待烧录完成

5. 烧录完成后**先不要给 ESP32 上电**，等服务端启动后再上电

cd /Users/mandy/Documents/GitHub/AI-bot/server

## 第六步：确认 API Key

服务端需要 LLM 的 API Key。检查 `server/.env` 文件里是否有：

```
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxx
```

如果没有，去 [openrouter.ai](https://openrouter.ai) 注册获取 Key，写入 `.env` 文件。

---

## 第七步：安装并启动 WhatsApp Bridge

打开 **终端1**：

```bash
cd /Users/mandy/Documents/GitHub/AI-bot/nanobot-src/bridge
npm install
npm run build
npm start
```

**首次启动**：终端会显示一个 QR 码。用手机 WhatsApp 扫码：

> WhatsApp → 设置 → 已关联的设备 → 关联设备 → 扫描终端上的 QR 码

扫码成功后终端会显示 `Connected` 之类的提示。**这个终端保持运行不要关闭。**

之后重启不需要再扫码（有本地 session 缓存）。

## 第八步：启动服务端

打开 **终端2**：

```bash
cd /Users/mandy/Documents/GitHub/AI-bot/server
python main.py
```

正常启动后应该看到类似输出：

```
WhatsApp channel 已启用，bridge: ws://localhost:3001
服务已启动: http://0.0.0.0:8765
WebSocket: ws://localhost:8765/ws/device
统一 outbound 消费者已启动

```

如果看到 `API Key 未设置` 的错误，回第六步检查 `.env`。

**这个终端也保持运行不要关闭。**

## 第九步：激活 WhatsApp 转发

**重要**：你需要先用手机 WhatsApp **给自己发一条消息**，让服务端记住你的 WhatsApp ID。

方法：用另一个手机/WhatsApp 给扫码的那个 WhatsApp 号发一条消息（随便发什么，比如"你好"）。

或者：如果你只有一个手机，让朋友给你的 WhatsApp 发一条消息。

服务端终端应该会显示收到 WhatsApp 消息的日志。之后设备的 AI 回复就会自动转发到这个 WhatsApp 对话。

---

## 第十步：给 ESP32 上电

1. 确认终端1（WhatsApp Bridge）和终端2（服务端）都在运行
2. 拨动 ESP32 电源开关上电（或 USB 连接电脑）
3. 观察 ESP32 屏幕变化：
   - `Starting...` → 初始化中
   - `WiFi: Connected` → WiFi 连接成功
   - `Server: Connected` → WebSocket 连接成功
4. 连接成功后，ESP32 会自动发送一条消息请求 AI 自我介绍
5. 几秒后屏幕会显示 AI 的回复文字
6. 同时 WhatsApp 上也会收到同样的回复

---

## 第十一步：开始语音交互

1. **用手指按住** ESP32 触摸区域（IO7 铜皮），屏幕显示 `Recording...`
2. **对着麦克风说话**，比如"帮我打开微信"
3. **松开手指**，屏幕显示 `Processing...`
4. 等待几秒，AI 回复会：
   - 显示在 ESP32 屏幕上
   - 同时出现在 WhatsApp 对话里
   - 如果 AI 决定执行电脑命令（比如打开微信），命令会在电脑上执行

## 常见问题排查

### ESP32 屏幕显示 WiFi: FAILED

- 检查 WiFi 名称和密码是否正确（`EE3070_P1615_1` / `EE3070P1615`）
- 确认 WiFi 热点已开启
- ESP32 会自动重启重试

### ESP32 屏幕显示 Server: Disconnected

- 检查电脑 IP 是否正确（固件里的 `WS_HOST`）
- 确认服务端正在运行（终端2）
- 确认电脑和 ESP32 在同一个 WiFi
- Mac 首次运行可能弹出防火墙提示，点"允许传入连接"

### WhatsApp 收不到消息

- 确认 WhatsApp Bridge 在运行（终端1）
- 确认已完成第九步（先给 WhatsApp 发一条消息激活转发）
- 检查终端2日志是否有"回复已转发到 WhatsApp"

### 录音后没有回复

- 录音太短（<0.5秒）会被忽略，按住久一点
- 检查终端2日志看 ASR 是否识别成功
- 检查 API Key 是否有效

### 终端1 QR 码扫不了

- 确保 Node.js 版本 >= 20（`node --version` 检查）
- 删除 bridge 目录下的 session 缓存文件夹，重新运行 `npm start`

## 启动顺序速查

```
1. 电脑连 WiFi: EE3070_P1615_1
2. 终端1: cd nanobot-src/bridge && npm start     （首次扫QR码）
3. 终端2: cd server && python main.py             （看到"服务已启动"）
4. 手机:  给 WhatsApp 发一条消息激活转发
5. ESP32: 上电 → 屏幕显示"Connected" → 自动自我介绍
6. 演示:  按住触摸 → 说话 → 松开 → 等回复
```
