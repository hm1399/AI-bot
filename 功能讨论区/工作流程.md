# 硬件上电后的联调流程：主流程不依赖 WhatsApp，只运行 `main.py`

> 更新时间：2026-04-08
>
> 当前主流程已经收口成：
> `硬件插电 -> 连接局域网 -> 按硬件触摸键 -> 电脑开始录音 -> 后端处理 -> 硬件喇叭播报`
>
> WhatsApp 现在只是可选辅助通道，不再是启动主流程的前置条件。

---

## 一、现在到底要跑哪条链路

默认主链路只有这 6 步：

1. 硬件上电，自动连你已经配置好的 WiFi
2. 电脑启动后端：`python3 main.py`
3. 硬件连上服务端
4. 你按住硬件触摸键
5. `main.py` 进程直接调用电脑本机麦克风录音
6. 你松开触摸键后，后端做 `ASR -> Agent -> TTS`，再把回复发回硬件喇叭和屏幕

一句话记住：

- 现在主流程只依赖 `main.py + 硬件`
- `server/tools/desktop_voice_client.py` 不是主流程
- WhatsApp 也不是主流程

---

## 二、启动前只确认 4 件事

### 1. 电脑和硬件在同一个局域网

- 电脑连接到硬件已经配置好的那个 WiFi
- 硬件插电后也会自动连接同一个 WiFi

### 2. 后端依赖已经装好

```bash
cd /Users/mandy/Documents/GitHub/AI-bot/server
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

如果 `sounddevice` 在 macOS 安装失败，先装：

```bash
brew install portaudio
```

### 3. `server/.env` 或 `config.yaml` 里有主流程需要的配置

至少要保证这些能用：

- `OPENROUTER_API_KEY` 或你实际使用的模型 API Key
- `APP_AUTH_TOKEN`
- `device.auth_token` 如果你当前设备联调需要鉴权

注意：

- `whatsapp.enabled` 现在默认是 `false`
- 不开 WhatsApp，不影响硬件语音主链路

### 4. 第一次录音时，系统可能弹麦克风权限

因为现在是 `python3 main.py` 直接代管电脑麦克风，所以首次按住硬件触摸键时：

- macOS 可能会弹出麦克风权限
- 要允许你当前运行 Python/Terminal 的进程访问麦克风

如果你点了拒绝，后面按键会触发，但电脑不会真正开始录音。

## 四、你实际演示时怎么操作

### 1. 按住硬件触摸键

按住以后，硬件会发送：

- `touch_event.long_press`

它现在不再本地采麦克风，只负责触发和状态反馈。

### 2. 电脑开始录音

只要链路正常，`main.py` 里的内置桌面麦克风服务会立刻开始采音。

你的动作就是：

- 手按住硬件
- 嘴对着电脑麦克风说

### 3. 松开硬件触摸键

松开以后，硬件发送：

- `touch_event.long_release`

随后服务端结束录音，并开始：

1. `ASR` 转文字
2. `Agent` 生成回复
3. `TTS` 合成音频
4. 通过设备通道把文本和 PCM 音频送回硬件

### 4. 硬件播报回复

链路正常时，你会看到或听到这些结果：

- 屏幕状态从 listening -> transcribing/process -> speaking -> idle
- 屏幕显示 AI 回复文本
- 硬件喇叭播放 TTS 回复

## 七、如果不通，先看哪里

### 1. 按键有反应，但电脑没录音

优先检查：

- macOS 麦克风权限有没有给当前 Python/Terminal
- `sounddevice` 有没有安装成功
- `main.py` 日志里是否出现桌面麦克风报错

### 2. 录音成功，但硬件没声音

优先检查：

- 功放和喇叭接线
- 板子是否刷的是支持 `audio_play + PCM + audio_play_end` 的新 `demo.ino`
- 服务端是否已经走到 TTS 返回阶段

### 3. 硬件没连上服务端

优先检查：

- 电脑和硬件是不是同一个 WiFi
- 电脑防火墙有没有拦 Python
- 后端是不是已经正常启动

## 八、如果你想额外打开 WhatsApp

WhatsApp 现在是可选项，只在你需要“消息镜像/辅助查看”时再开。

### 1. 打开配置

把 `server/config.yaml` 里的：

```yaml
whatsapp:
  enabled: false
```

改成：

```yaml
whatsapp:
  enabled: true
```

### 2. 启动 bridge

```bash
cd /Users/mandy/Documents/GitHub/AI-bot/server/bridge
npm install
npm run build
npm start
```

### 3. 扫二维码

手机 WhatsApp 路径：

- `设置`
- `已关联的设备`
- `关联设备`
- 扫终端里的二维码

### 4. 再启动 `main.py`

这时 WhatsApp 会作为辅助通道在线：

- 不影响硬件主链路
- 只是在硬件回复之外，多一个 WhatsApp 同步消息入口
