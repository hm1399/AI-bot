# AI-Bot 硬件测试计划

## 测试顺序原则

按照**电源 → 主控 → 外设**的顺序，从最基础的开始，每一步确认OK后再进入下一步。每个测试都是独立的 Arduino sketch。

---

## 测试1：串口通信（已完成 ✅）

**目的：** 确认 ESP32-S3 通过 USB CDC 与电脑正常通信

**测试内容：**
- USB Type-C 连接电脑，串口监视器 115200 波特率
- 启动信息打印
- 回显测试（输入什么返回什么）
- 心跳包（每5秒打印运行时间）

**判定标准：** 串口监视器能看到输出，回显正常

**文件：** `firmware/arduino/sketch_mar7a/test.ino`

## 测试2：屏幕显示（进行中 🔄）

**目的：** 确认 ST7789 1.54寸 SPI 屏幕能正常显示

**引脚：** MOSI=IO11, SCK=IO12, CS=IO9, DC=IO10, RST=IO46

**测试内容：**
- 红、绿、蓝纯色填充（确认屏幕能亮、颜色正确）
- 文字显示
- 几何图形绘制

**判定标准：** 屏幕依次显示三种颜色，文字和图形清晰可见

**常见问题：**
- 屏幕全黑：检查 FPC 排线是否插紧、背光供电是否正常
- 颜色反转：调整 TFT_INVERSION_ON / OFF
- 显示偏移：调整 offset 参数

**文件：** `firmware/arduino/test2/test2.ino`

**依赖库：** TFT_eSPI（需配置 User_Setup.h）

## 测试3：麦克风（INMP441）（已完成 ✅）

**目的：** 确认 I2S 麦克风能采集到声音数据

**引脚：** SCK=IO14, WS=IO15, SD=IO16

**测试内容：**
- I2S 初始化，16kHz 采样率，16bit
- 读取音频数据，串口打印采样值
- 静音时数据应接近0，说话时数值有明显波动
- 可选：计算分贝值，串口打印音量条

**判定标准：** 静音时采样值稳定在低位，对着麦克风说话时数值有大幅变化

**常见问题：**
- 全是0：检查 I2S 引脚接线、L/R 选择电阻 R12 (100kΩ下拉=左声道)
- 数据不变：确认 WS 和 SCK 没接反

---

## 测试4：功放与喇叭（MAX98357A）

**目的：** 确认 I2S 功放能驱动喇叭发声

**引脚：** BCLK=IO17, LRC=IO18, DIN=IO8

**测试内容：**
- I2S 输出正弦波（440Hz，A4音符）
- 播放不同频率的蜂鸣声
- 音量测试（调整数据幅值）

**判定标准：** 喇叭能发出清晰的正弦波蜂鸣声

**常见问题：**
- 无声音：检查喇叭焊接、BCLK/LRC/DIN 接线
- 声音有噪音：检查 MAX98357A 的 VBAT 去耦电容 C7
- GAIN 引脚悬空 = 9dB 增益（默认）

---

## 测试5：麦克风回环测试（录放一体）

**目的：** 验证麦克风采集 + 功放播放的完整音频链路

**测试内容：**
- 麦克风实时采集音频数据
- 直接通过 I2S 输出到功放
- 对着麦克风说话，喇叭实时回放

**判定标准：** 喇叭能实时回放麦克风采集到的声音，延迟可接受

**依赖：** 测试3 + 测试4 都通过

## 测试6：触摸感应（已完成 ✅）

**目的：** 确认 ESP32-S3 内置电容触摸功能正常

**引脚：** IO7 (TOUCH_MAIN)

**测试内容：**
- 读取触摸引脚原始值，串口打印
- 设定阈值，检测触摸/释放事件
- 实现单击、双击、长按三种手势识别

**判定标准：** 触摸铜皮时串口输出触摸事件，松开后恢复

**常见问题：**
- 触摸不灵敏：调整阈值、检查触摸子板焊接和 ESD 保护电阻 R5
- 误触发：提高阈值或增加去抖延时

## 测试7：MPU6050 六轴传感器（已完成 ✅）

**目的：** 确认 I2C 通信和传感器数据读取正常

**引脚：** SDA=IO5, SCL=IO6, INT=IO4

**测试内容：**
- I2C 扫描，确认能找到 MPU6050 地址（0x68）
- 读取加速度 XYZ 和陀螺仪 XYZ 原始数据
- 平放时 Z 轴加速度应接近 ±16384（1g）
- 摇一摇检测（加速度突变）

**判定标准：** I2C 扫描到设备，倾斜/摇晃时数据有对应变化

**依赖库：** Wire.h（内置）

**常见问题：**
- 扫描不到设备：检查 SDA/SCL 接线、模组供电
- 数据全是0或-1：I2C 地址错误，检查 AD0 引脚电平

## 测试8：WS2812B RGB 灯带

**目的：** 确认 LED 灯带控制正常

**引脚：** IO38 (LED_DIN)

**测试内容：**
- 逐个点亮红、绿、蓝
- 彩虹渐变效果
- 呼吸灯效果

**判定标准：** LED 灯依次显示正确颜色，动画流畅

**依赖库：** Adafruit_NeoPixel 或 FastLED

**常见问题：**
- 不亮：检查数据线方向（DIN 不是 DOUT）、供电
- 颜色错误：GRB/RGB 顺序设置

## 测试9：WiFi 连接（已完成 ✅）

**目的：** 确认 WiFi 功能正常，能连接路由器

**测试内容：**
- 扫描周围 WiFi 热点，串口打印 SSID 列表
- 连接指定 WiFi（硬编码 SSID 和密码）
- 打印获取到的 IP 地址
- HTTP GET 请求测试网络连通性

**判定标准：** 成功连接 WiFi，获取 IP，HTTP 请求返回 200

**常见问题：**
- 连接失败：检查天线区域是否被铜皮覆盖（PCB 设计时需注意净空区）
- 信号弱：ESP32-S3 模组自带 PCB 天线，确保天线区无遮挡

---

## 测试10：电源与充电（已完成 ✅）

**目的：** 确认电源管理电路工作正常

**测试内容（万用表手动测量）：**
- USB 插入：TP4056 充电指示灯 D1（红）亮起
- 充满后：D2（绿）亮起，D1 熄灭
- SW1 拨动开关：拨到 ON，AMS1117 输出 3.3V ±0.1V
- 拨到 OFF，3.3V 输出为 0V
- 电池供电运行：拔掉 USB，仅靠电池供电，系统正常工作

**判定标准：** 充电指示灯正确、3.3V 稳压输出准确、电池独立供电正常

**注意：** 此测试主要用万用表，不需要写代码。可选写一个读取电池电压的 ADC 程序。
