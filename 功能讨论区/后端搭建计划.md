## Phase 4: 语音交互串联 ✅

**目标：** 把 ASR + TTS 串联进 DeviceChannel，实现完整的语音交互链路。

### 4.1 音频接收 + ASR 集成

- [x] DeviceChannel 中实现音频帧接收：
  - 收到二进制帧 → 累积到 `self.audio_buffer`（`bytearray`）
  - 收到 `{"type": "audio_end"}` → 触发识别流程
- [x] 识别流程：
  ```python
  async def _on_audio_end(self):
      pcm_data = bytes(self.audio_buffer)
      self.audio_buffer.clear()
      text = await self.asr.transcribe(pcm_data)
      if text.strip():
          # 构造 InboundMessage 发给 AgentLoop
          await self.bus.inbound.put(InboundMessage(..., content=text))
  ```

### 4.2 TTS 集成 + 音频发送

- [x] DeviceChannel 的 `send()` 方法增加 TTS 流程：
  ```python
  async def send(self, message: OutboundMessage):
      text = message.content
      # 1. 发屏幕显示
      await self._send_json({"type": "display_update", "data": {"text": text}})
      # 2. 发状态切换 → SPEAKING
      await self._send_json({"type": "state_change", "data": {"state": "SPEAKING"}})
      # 3. TTS 合成并流式发送音频
      await self._send_json({"type": "audio_play", "data": {}})
      async for chunk in self.tts.synthesize_stream(text):
          await self.ws.send_bytes(chunk)
      await self._send_json({"type": "audio_play_end", "data": {}})
      # 4. 恢复 IDLE
      await self._send_json({"type": "state_change", "data": {"state": "IDLE"}})
  ```

### 4.3 音频格式处理

- [x] 确认 ESP32 发送的 PCM 格式与 Whisper 期望一致
- [x] 确认 TTS 输出的 PCM 格式与 ESP32 播放期望一致
- [x] 处理字节序问题（ESP32 little-endian）

### 4.4 延迟优化

- [x] 测量各环节耗时（ASR / LLM / TTS），打印日志
- [x] TTS 流式发送：不等合成完毕，边合成边发，减少首音延迟
- [ ] 考虑 LLM 流式响应（streaming）：Claude 边生成边触发 TTS（可选，MVP 先不做）
