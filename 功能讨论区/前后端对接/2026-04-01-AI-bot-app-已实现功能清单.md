## 已实现功能

### 1. 目录层

- 已将前端统一放入 `AI-bot/app`
- 当前后端目录边界已满足：
  - 前端目录：`app/`
  - 后端目录：`server/`

### 2. 协议层

- HTTP 基础前缀已切到 `/api/app/v1/...`
- WebSocket 已切到 `/ws/app/v1/events`
- 已统一解析后端 envelope：
  - 成功：`{ ok, data, request_id, ts }`
  - 失败：`{ ok: false, error: { code, message }, request_id, ts }`
- 已支持 HTTP 鉴权：
  - `Authorization: Bearer <token>`
  - `X-App-Token: <token>`
- 已支持 WebSocket 鉴权与恢复参数：
  - `token`
  - `last_event_id`
  - `replay_limit`
- 已将 `404/501/NOT_IMPLEMENTED` 统一识别为后端未就绪状态

### 3. Connect

- 已支持手动输入 `host`
- 已支持手动输入 `port`
- 已支持可选 `App Token`
- 已接入 `GET /api/health`
- 已接入 `GET /api/app/v1/bootstrap`
- 已保存本地连接信息：
  - `host`
  - `port`
  - `token`
  - `current_session_id`
  - `latest_event_id`
- 已保留 Demo Mode
- 已移除旧的假网络扫描逻辑，并明确降级为未来能力

### 4. WebSocket 与启动恢复

- 已接入 `/ws/app/v1/events`
- 已保存最后一个 `event_id`
- 已在重连时带上 `last_event_id`
- 已实现自动重连
- 已处理 `system.hello`
- 当服务端要求 `should_refetch_bootstrap` 时，已触发重新拉取 bootstrap

### 5. Chat

- 已接入：
  - `GET /api/app/v1/sessions`
  - `GET /api/app/v1/sessions/{session_id}/messages`
  - `POST /api/app/v1/sessions/{session_id}/messages`
- 已把 `accepted_message` 渲染为 pending
- 已处理事件：
  - `session.message.created`
  - `session.message.progress`
  - `session.message.completed`
  - `session.message.failed`
- AI 回复已改成仅来自后端事件流
- 前端代码中未发现直连 OpenAI / Anthropic / Gemini 的请求逻辑
- 语音输入入口已保留，但当前仍是入口占位，不是完整采集链路

### 6. Home / Dashboard

- 已接入 `GET /api/app/v1/runtime/state`
- 已接入 `GET /api/app/v1/device`
- 已展示：
  - 当前任务
  - 任务队列长度
  - 设备快照
  - Todo 摘要
  - Calendar 摘要
- 已处理事件：
  - `runtime.task.current_changed`
  - `runtime.task.queue_changed`
  - `device.connection.changed`
  - `device.state.changed`
  - `device.status.updated`
  - `todo.summary.changed`
  - `calendar.summary.changed`
- 已保留与当前后端契约一致的快捷动作：
  - `speak`
  - `stop`
  - `sync`

### 7. Settings

- 设置页已改为后端驱动
- 已接入：
  - `GET /api/app/v1/settings`
  - `PUT /api/app/v1/settings`
  - `POST /api/app/v1/settings/llm/test`
- 已将“测试 AI 连接”改为后端接口调用
- 当前实现不再在前端持久化第三方模型 SDK client
- 当前实现保留 provider/model 等字段，但语义是“编辑后端配置”，不是前端直连模型

### 8. Tasks / Events

- 已定义任务与事件 DTO 映射
- 已定义对应 service 调用入口
- 后端未提供时，会进入统一“未就绪”状态
- 页面会保留入口和按钮位置，并给出明确提示，不会静默失败

### 9. Demo Mode

- Demo Mode 已保留
- Demo 与真实后端模式已经分离
- 页面层主要消费统一模型，而不是直接把 mock 结构散落到页面里

### 10. 当前验证结果

- 已在 `AI-bot/app` 下重新执行 `npm run build`
- 构建通过

## 未实现或与计划不一致的部分

### 1. 不是标准 Flutter 工程

计划要求的结构是：

- `app/android`
- `app/ios`
- `app/web`
- `app/assets`
- `app/lib`
- `app/test`
- `app/pubspec.yaml`

当前实际仍是 React/Vite 工程结构：

- `app/src`
- `app/package.json`
- `app/vite.config.ts`

所以“Flutter 工程骨架已完成”这一项不能判定为完成。

### 2. Control Center 未独立实现

- 计划中的 `Control Center` 独立页面当前不存在
- 目前相关能力被吸收到 `Home` 页面的一部分

### 3. Notifications / Reminders 未实现

- 当前 `app/src/app` 下没有 notifications/reminders 对应的页面、context 或 service
- 这部分还未达到“已写好占位与接口层”的完成度

### 4. 语音输入仍是入口占位

- 目前只是保留了按钮入口和提示
- 未实现真正的本地采集、转写、上传后端的完整链路

### 5. 自动化测试未补齐

- 计划中要求的错误映射测试、聊天状态测试、断线重连测试、service 级测试目前都没有落地
- `package.json` 中也没有测试脚本

### 6. Demo Mode 依赖注入层未按 Flutter 计划成型

- 当前是 React Context 层面的 real/demo 分支
- 还不是计划中 `demo_service_bundle + app_providers` 那种明确的依赖注入结构

### 7. 接入文档与运行文档未完成专项收尾

- 虽然 `app/` 中有现成说明文件
- 但尚未形成一份严格对应 `AI-bot/server app-v1` 契约的新收尾文档

### 8. 存在一个明显的文案编码问题

- `Settings.tsx` 中有一段“后端设置接口尚未提供...”的中文提示出现乱码
- 不影响主流程，但属于需要修复的质量问题
