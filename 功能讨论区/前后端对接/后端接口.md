> 更新时间：2026-04-01  
> 适用后端：`AI-bot/server` 当前实现  
> 目标读者：Flutter 前端对接同学

---

## 1. 基础约定

### 1.1 Base Path

- HTTP: `/api/app/v1`
- WebSocket: `/ws/app/v1/events`

### 1.2 鉴权

HTTP 二选一：

- `Authorization: Bearer <token>`
- `X-App-Token: <token>`

WebSocket：

- `GET /ws/app/v1/events?token=<token>`

断线续传参数：

- `last_event_id`
- `replay_limit`

### 1.3 成功响应

```json
{
  "ok": true,
  "data": {},
  "request_id": "req_xxx",
  "ts": "2026-04-01T12:00:00+08:00"
}
```

### 1.4 失败响应

```json
{
  "ok": false,
  "error": {
    "code": "INVALID_ARGUMENT",
    "message": "content is required"
  },
  "request_id": "req_xxx",
  "ts": "2026-04-01T12:00:00+08:00"
}
```

### 1.5 通用规则

- JSON 字段统一 `snake_case`
- 时间统一 ISO-8601 且带时区
- 所有 ID 都是字符串

## 2. 已有可直接复用接口

### `GET /api/health`

用途：服务健康检查。

### `GET /api/app/v1/bootstrap`

用途：Flutter 启动后的唯一初始化入口。

返回重点字段：

- `server_version`
- `capabilities`
- `runtime`
- `sessions`
- `event_stream.path`
- `event_stream.resume.latest_event_id`

### `GET /api/app/v1/sessions`

用途：获取会话列表。

支持 query：

- `limit`
- `archived`
- `pinned_first`

### `POST /api/app/v1/sessions`

用途：创建会话。

请求：

```json
{
  "title": "新对话"
}
```

### `GET /api/app/v1/sessions/{session_id}`

用途：获取单个会话信息。

### `GET /api/app/v1/sessions/{session_id}/messages`

用途：获取消息列表。

支持 query：

- `limit`
- `before`
- `after`

### `POST /api/app/v1/sessions/{session_id}/messages`

用途：发送用户消息。

请求：

```json
{
  "content": "你好",
  "client_message_id": "flutter_local_001"
}
```

成功返回：

```json
{
  "accepted_message": {
    "message_id": "msg_xxx",
    "session_id": "app:main",
    "role": "user",
    "content": "你好",
    "content_type": "text",
    "status": "pending",
    "created_at": "2026-04-01T12:00:00+08:00",
    "metadata": {
      "client_message_id": "flutter_local_001",
      "task_id": "task_xxx"
    }
  },
  "task_id": "task_xxx",
  "queued": true
}
```

### `GET /api/app/v1/runtime/state`

用途：获取共享运行态快照。

### `POST /api/app/v1/runtime/stop`

用途：停止当前任务，或按 `task_id` 停止指定任务。

### `GET|POST /api/app/v1/runtime/todo-summary`

用途：读取 / 更新 Todo 摘要。

### `GET|POST /api/app/v1/runtime/calendar-summary`

用途：读取 / 更新日历摘要。

### `GET /api/app/v1/device`

用途：获取设备状态快照。

### `POST /api/app/v1/device/speak`

用途：让设备直接播报一段文字。

请求：

```json
{
  "text": "你好，这是测试播报"
}
```

### `GET /api/app/v1/capabilities`

用途：获取能力开关。

当前新增返回字段包括：

- `settings`
- `tasks`
- `events`
- `notifications`
- `reminders`
- `device_commands`

### `GET /ws/app/v1/events`

用途：订阅实时事件流。

## 3. 新增接口

## 3.1 Settings

### `GET /api/app/v1/settings`

用途：读取当前设置页数据。

返回：

```json
{
  "server_url": "0.0.0.0",
  "server_port": 8765,
  "llm_provider": "openrouter",
  "llm_model": "z-ai/glm-4.7",
  "llm_base_url": null,
  "llm_api_key_configured": true,
  "stt_provider": "funasr",
  "stt_model": "FunAudioLLM/SenseVoiceSmall",
  "stt_language": "auto",
  "tts_provider": "edge_tts",
  "tts_model": "edge_tts",
  "tts_voice": "zh-CN-XiaoxiaoNeural",
  "tts_speed": 1.0,
  "device_volume": 75,
  "led_enabled": true,
  "led_brightness": 50,
  "led_mode": "breathing",
  "led_color": "#0000ff",
  "wake_word": "Hey Assistant",
  "auto_listen": true
}
```

注意：

- 不会返回原始 `llm_api_key`
- 只返回 `llm_api_key_configured`
- 当前实现采用 runtime overlay 持久化，不会直接改仓库里的 `server/config.yaml`

### `PUT /api/app/v1/settings`

用途：保存设置页。

支持部分更新。

请求示例：

```json
{
  "llm_provider": "openai",
  "llm_model": "gpt-4o",
  "llm_api_key": "new-secret",
  "llm_base_url": null,
  "stt_language": "en-US",
  "tts_voice": "alloy",
  "tts_speed": 1.0,
  "device_volume": 75,
  "led_mode": "breathing",
  "led_color": "#0000ff",
  "wake_word": "Hey Assistant",
  "auto_listen": true
}
```

返回：更新后的公开设置对象。

### `POST /api/app/v1/settings/llm/test`

用途：测试当前 LLM 配置连通性。

请求体可为空，也可带临时覆盖字段；若带了不会自动保存。

成功返回：

```json
{
  "success": true,
  "provider": "openai",
  "model": "gpt-4o",
  "message": "connection ok"
}
```

错误码：

- `INVALID_ARGUMENT`
- `LLM_NOT_CONFIGURED`
- `UPSTREAM_AUTH_FAILED`
- `UPSTREAM_TIMEOUT`

## 3.2 Tasks

### `GET /api/app/v1/tasks`

用途：任务列表页。

支持 query：

- `completed=true|false`
- `priority=high|medium|low`
- `limit`

返回：

```json
{
  "items": [
    {
      "task_id": "task_001",
      "title": "Review proposal",
      "description": null,
      "priority": "high",
      "completed": false,
      "due_at": null,
      "created_at": "2026-04-01T10:00:00+08:00",
      "updated_at": "2026-04-01T10:00:00+08:00"
    }
  ]
}
```

### `POST /api/app/v1/tasks`

请求：

```json
{
  "title": "Review proposal",
  "description": "Prepare feedback",
  "priority": "high",
  "completed": false,
  "due_at": "2026-04-02T10:00:00+08:00"
}
```

返回：新创建的 task 对象。

### `PATCH /api/app/v1/tasks/{task_id}`

支持修改：

- `completed`
- `priority`
- `title`
- `description`
- `due_at`

返回：更新后的 task 对象。

### `DELETE /api/app/v1/tasks/{task_id}`

返回：

```json
{
  "deleted": true,
  "task_id": "task_001"
}
```

事件：

- `task.created`
- `task.updated`
- `task.deleted`

事件 payload：

```json
{
  "task": {
    "task_id": "task_001"
  }
}
```

## 3.6 Device Commands

### `POST /api/app/v1/device/commands`

用途：统一设备控制命令入口。

请求：

```json
{
  "client_command_id": "cmd_local_001",
  "command": "set_volume",
  "params": {
    "level": 80
  }
}
```

允许的 `command`：

- `mute`
- `toggle_led`
- `restart`
- `wake`
- `sleep`
- `set_volume`
- `set_led_color`
- `set_led_brightness`

参数约定：

- `set_volume`: `params.level`，`0-100`
- `set_led_brightness`: `params.level`，`0-100`
- `set_led_color`: `params.color`，非空字符串
- 其他命令当前不要求参数

成功返回：

```json
{
  "accepted": true,
  "command_id": "cmd_srv_001",
  "command": "set_volume",
  "device": {
    "connected": true,
    "state": "IDLE",
    "battery": 85,
    "wifi_rssi": -52,
    "charging": false,
    "reconnect_count": 0
  }
}
```

错误码：

- `DEVICE_OFFLINE`
- `INVALID_ARGUMENT`
- `COMMAND_NOT_SUPPORTED`

当前后端会把命令转成设备侧 `device_command` 消息发送给 ESP32。

## 6. 当前实现备注

- `settings`、`tasks`、`events`、`notifications`、`reminders` 当前都持久化在 `server/workspace/runtime/` 下的 JSON 文件。
- `llm_api_key` 单独存储，不会通过接口回传。
- `notifications` 目前只开放读取/已读/删除接口，不开放前端手工创建接口。
- `reminders` 目前只有 CRUD，还没有真正调度执行。
- `GET|POST /api/app/v1/voice/commands` 当前没有实现，不属于这轮接口范围。
