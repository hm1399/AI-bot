# 故障排除指南

## ✅ 错误已修复

以下错误现已修复：
- ❌ `WebSocket error: { "isTrusted": true }` 
- ❌ `TypeError: Load failed`

### 修复内容

1. **防止启动时自动连接**
   - 应用启动时不再尝试连接 WebSocket
   - 只在用户点击"Connect"后才建立连接

2. **添加连接检查**
   - 所有 API 调用前检查是否已设置服务器地址
   - 所有数据获取前检查 WebSocket 是否已连接

3. **改进错误处理**
   - 所有网络请求都有 try-catch 保护
   - 错误只记录到控制台，不会中断应用

## ⚠️ 仍然可能出现的情况

### 情况 1: 连接失败
**现象：** 点击 Connect 后仍显示 "Disconnected"

**原因：**
- 后端服务器未启动
- IP 地址或端口错误
- 防火墙阻止连接

**解决：**
1. 确认后端服务器正在运行
2. 检查 IP 和端口是否正确
3. 查看浏览器控制台的具体错误信息

### 情况 2: 自动断开
**现象：** 连接后又变回 "Disconnected"

**原因：**
- 后端服务器关闭
- 网络不稳定
- WebSocket 端点配置错误

**解决：**
1. 检查后端服务器日志
2. 确认 WebSocket 端点：`ws://IP:PORT/ws/app`
3. 应用会在 5 秒后自动重连

### 情况 3: API 调用失败
**现象：** Toast 提示 "Failed to ..."

**原因：**
- 后端 API 端点未实现
- 后端返回错误
- 网络问题

**解决：**
1. 查看浏览器 Network 标签页
2. 检查后端日志
3. 确认 API 端点已实现

## 🔄 重置应用

如果一切都不工作，尝试：

1. **清除浏览器缓存**
   ```
   Ctrl + Shift + Delete
   选择 "Cached images and files"
   点击 "Clear data"
   ```

2. **硬刷新页面**
   ```
   Ctrl + Shift + R (Windows/Linux)
   Cmd + Shift + R (Mac)
   ```

3. **重新启动开发服务器**
   ```bash
   # 停止服务器 (Ctrl + C)
   # 然后重新启动
   pnpm run dev
   ```

4. **检查后端服务器**
   ```bash
   # 重启 Python 后端
   python main.py
   ```
