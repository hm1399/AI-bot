# ESP32-S3 原理图设计教程（嘉立创EDA）

> 面向零基础用户，一步步教你在嘉立创EDA中画出ESP32-S3的最小系统原理图。
> 每一步不仅告诉你"怎么接"，还解释"为什么这么接"和"还有什么别的接法"。

---

## 第一步：打开嘉立创EDA

1. 打开浏览器，访问 `https://lceda.cn` （嘉立创EDA专业版）
2. 注册/登录嘉立创账号
3. 点击左上角 **"文件" → "新建" → "工程"**
4. 工程名填：`AI-Bot桌面助手`，点击确定
5. 在工程里 **右键 → "新建" → "原理图"**，名称填 `主控电路`

> 你现在看到的就是原理图编辑界面：中间是白色画布，左边是元件库，上面是工具栏。

---

## 第二步：理解"原理图"是什么

原理图不是实物图，它是一张**逻辑连接图**，告诉你：

- 哪些引脚连在一起
- 每个芯片需要哪些外围电路（电容、电阻等）
- 电源怎么分配

你可以把它想象成"电路的设计蓝图"，后面做PCB板就是根据这个蓝图来的。

## 第三步：放置ESP32-S3模块

### 为什么用"模组"而不是"裸芯片"？

ESP32-S3裸芯片是QFN56封装（7x7mm），引脚很小很密，需要自己设计天线、晶振、flash等，非常复杂。

**新手建议直接用ESP32-S3模组**（ESP32-S3-WROOM-1-N16R8），模组里已经把天线、flash、晶振都集成好了，你只需要接电源和信号线。

> **另一种选择**：如果你对产品体积有极致要求，可以用裸芯片ESP32-S3自己搭最小系统。但需要设计40MHz晶振电路、RF匹配网络、PCB天线，还需要外挂SPI Flash芯片，难度非常高，不推荐新手。

### 操作步骤

1. 点击右侧 **"元件库"** 面板（或快捷键 `Shift+F`）
2. 在搜索框输入：`ESP32-S3-WROOM`
3. 你会看到搜索结果，选择 **嘉立创自营库** 中的元件（带蓝色"嘉"标志的优先）
4. 找到 `ESP32-S3-WROOM-1-N16R8`（16MB Flash + 8MB PSRAM，推荐）
5. 点击 **"放置"**，然后在画布中间点一下，模块就出现了

> **小提示**：如果搜不到，试试搜嘉立创商城编号 `C2913202`，或者直接搜 `ESP32-S3`。

## 第四步：认识ESP32-S3模组的引脚

放好模组后，你会看到一个方框，周围有很多引脚。别慌，我们只关心这几类：

| 引脚类型 | 引脚名 | 你需要做什么 |
|---------|--------|-------------|
| **电源** | 3V3 | 接3.3V电源 |
| **地** | GND | 接地（通常有多个，全部接地） |
| **使能** | EN | 接上拉电阻+复位按键 |
| **启动模式** | GPIO0 | 接上拉电阻+烧录按键 |
| **USB** | GPIO19(D-), GPIO20(D+) | 接USB口（用于烧录和调试） |
| **通用IO** | GPIO1~GPIO48 | 后面接麦克风、功放、屏幕等 |

## 第五步：画电源部分（最重要！）

### 5.1 为什么电源最重要？

芯片就像一个需要喝水的人——如果水（电源）不干净、忽大忽小，人就会生病（芯片就会工作异常、死机、甚至烧毁）。电源部分的作用就是给芯片提供"干净、稳定的饮用水"。

### 5.2 放置电源符号

1. 在元件库搜索 `VCC_3.3V` 或直接用电源端口工具
2. 放置一个 **3.3V** 电源符号，连到模组的 `3V3` 引脚
3. 搜索 `GND`，放置地符号，连到模组所有的 `GND` 引脚

**为什么GND要全部接？** GND就是电流的"回家路"。如果有的GND没接，电流回不去，会从其他路径乱跑，产生干扰。模组上所有GND引脚在芯片内部并不是完全连通的（有的是电源地，有的是射频地），全接上才能保证各部分的地电位一致。

### 5.3 添加去耦电容

#### 去耦电容是什么？为什么要加？

想象一下：芯片工作时，一会儿在算数据（耗电多），一会儿在等待（耗电少）。这种忽高忽低的用电就像水龙头一开一关，会导致水管里的水压波动。

**去耦电容就是水管旁边的小蓄水池**：
- 芯片突然需要大电流 → 电容立刻放电补上，不用等远处的电源慢慢送过来
- 芯片突然不需要电流 → 电容把多余的能量存起来

没有去耦电容，芯片电源引脚上的电压会像心跳一样上下跳动（叫做"纹波"），严重时芯片会死机或重启。

#### 为什么放2个100nF + 1个10uF，而不是只放1个？

不同大小的电容擅长过滤不同频率的干扰：

| 电容值 | 擅长过滤的干扰 | 类比 |
|-------|--------------|------|
| 100nF (小) | 高频噪声（几MHz~几百MHz） | 像细网，过滤沙子 |
| 10uF (大) | 低频波动（几kHz~几MHz） | 像粗网，过滤石子 |

两种搭配使用，高频低频都能过滤，这是行业标准做法。放2个100nF是因为ESP32-S3内部有多个电源域（CPU核心、WiFi射频、IO等），多一个覆盖更全。

> **其他做法**：一些高要求的设计会放3~4个100nF + 1个22uF，甚至加一个1nF陶瓷电容过滤超高频。对于我们的项目，2个100nF + 1个10uF完全够用。

#### 操作

1. 在元件库搜索 `100nF`（或 `0.1uF`），选择 `0603` 封装的贴片电容
2. 放置 **2个 100nF 电容**，紧挨着模组放
3. 再放 **1个 10uF 电容**（搜索 `10uF`，选 `0603` 或 `0805` 封装）
4. 每个电容一端接 `3V3`，另一端接 `GND`

**为什么要"紧挨着模组放"？** 电容离芯片越近，效果越好。如果电容放得远，连接导线本身就有电感（阻碍电流快速通过），等电容的电送到芯片那边，干扰已经发生了。就像蓄水池要建在用水点旁边，而不是隔了一条街。

画完后看起来像这样（示意）：

```text
3V3 ──┬──┬──┬── 模组的3V3引脚
      │  │  │
    100nF 100nF 10uF     ← 这三个电容要尽量靠近模组
      │  │  │
GND ──┴──┴──┴── 模组的GND引脚
```

## 第六步：画EN引脚电路（使能/复位）

### EN引脚是什么？为什么需要处理它？

EN = Enable（使能），是芯片的"总开关"：
- EN = 高电平（3.3V） → 芯片正常工作
- EN = 低电平（0V） → 芯片关机/复位

如果EN引脚悬空不接任何东西（叫做"浮空"），它就像一根天线，会随机接收到周围的电磁干扰，导致电平忽高忽低，芯片会随机重启。所以**必须给它一个确定的电平**。

### 电路设计

需要的元件：

| 元件 | 值 | 封装 | 数量 | 用途 |
|------|---|------|------|------|
| 电阻 R1 | 10K | 0603 | 1 | EN上拉 |
| 电容 C4 | 100nF | 0603 | 1 | EN滤波 |
| 轻触开关 SW1 | - | - | 1 | RST复位按键 |

连接方式：

```text
         R1 (10K)
3V3 ────┤├──────┬─── EN引脚（模组）
                │
              C4 (100nF)
                │
               GND

同时：

EN引脚 ──── SW1(RST按键) ──── GND
```

### 每个元件为什么要加？

**R1 (10K电阻) — 上拉电阻：**

作用：把EN引脚"拉"到高电平(3.3V)，让芯片默认处于工作状态。

为什么用10K而不是其他值？
- 太小（比如100Ω）：按下RST时，3.3V通过100Ω到GND，电流 = 3.3V/100Ω = 33mA，白白浪费电，还可能损坏引脚
- 太大（比如1MΩ）：拉的力量太弱，容易被干扰打败，EN还是不稳定
- 10K是折中：按键按下时电流 = 3.3V/10K = 0.33mA，几乎不耗电，拉高的力量也足够

> **其他选择**：4.7K~100K都可以用，业界最常用的是10K。如果你的环境干扰很大（比如旁边有大功率电机），可以用4.7K，拉得更"狠"一点。

**C4 (100nF电容) — 滤波电容：**

作用：过滤EN引脚上的高频干扰，防止短暂的电磁脉冲误触发复位。

原理：电容有个特性——电压不能突变。如果有一个持续只有几纳秒的干扰脉冲想把EN拉低，电容会把这个脉冲"吸收"掉，EN电压不会有明显变化。只有你按下按键持续把EN拉低，电容才会慢慢放电，EN才真的变低。

还有一个副作用：这个电容会让芯片上电时EN缓慢上升（因为电容充电需要时间），起到**上电延迟**的效果。这在某些情况下反而是好事——让电源先稳定下来，芯片再启动。

> **其他选择**：
> - 不加电容：如果你的环境很干净、板子也不复杂，不加也能工作。但加了更稳妥，成本才几分钱。
> - 用1uF：延迟更长，芯片上电更慢，一般没必要。
> - 加一个二极管（比如1N4148）：反向并联在RST按键上，可以在芯片掉电时快速放电复位，用于需要快速重启的场景。我们的项目不需要。

**SW1 (轻触开关) — RST复位按键：**

作用：手动按下就能让芯片重启，相当于电脑的重启键。

工作原理：
- 不按时：按键断开，EN通过10K电阻被拉到3.3V → 芯片工作
- 按下时：按键导通，EN被直接连到GND → 芯片复位
- 松开后：EN又被拉回3.3V → 芯片重新启动

> **另一种做法**：不加物理按键，改用软件复位（通过代码或USB命令触发复位）。但开发调试阶段，物理按键方便很多——芯片卡死了按一下就好，不需要拔电源。强烈建议保留。

## 第七步：画GPIO0电路（烧录/启动模式）

### GPIO0是什么？为什么需要处理？

ESP32-S3启动时会"看"GPIO0的电平来决定进入哪种模式：

| GPIO0电平 | 启动模式 | 说明 |
|----------|---------|------|
| 高电平(3.3V) | **正常运行模式** | 从Flash读取程序并执行 |
| 低电平(0V) | **下载模式** | 等待通过USB/串口接收新固件 |

和EN一样，GPIO0如果浮空也会出问题（随机进入下载模式，芯片不执行程序）。

### 电路设计

需要的元件：

| 元件 | 值 | 封装 | 数量 | 用途 |
|------|---|------|------|------|
| 电阻 R2 | 10K | 0603 | 1 | GPIO0上拉 |
| 轻触开关 SW2 | - | - | 1 | BOOT烧录按键 |

连接方式：

```text
         R2 (10K)
3V3 ────┤├──────┬─── GPIO0引脚（模组）
                │
          SW2(BOOT按键)
                │
               GND
```

### 每个元件为什么要加？

**R2 (10K电阻) — 上拉电阻：**

让GPIO0默认为高电平，芯片上电后默认进入正常运行模式（执行你烧录好的程序）。

如果不加这个电阻，GPIO0浮空，芯片有时正常启动有时进下载模式，很不稳定。

> **为什么GPIO0不像EN那样再加一个滤波电容？** 因为GPIO0只在上电那一瞬间被芯片采样，之后它就变成普通GPIO了。如果加了电容，按下BOOT键后电容放电太慢，可能来不及在芯片采样时把电平拉到足够低，反而影响进入下载模式。

**SW2 (轻触开关) — BOOT烧录按键：**

作用：需要烧录新固件时，配合RST键使用。

操作流程：
1. **按住BOOT键**（GPIO0被拉低）
2. **按一下RST键再松开**（芯片复位重启）
3. **松开BOOT键**
4. 此时芯片已进入下载模式，电脑就可以烧录新固件了

> **另一种做法**：ESP32-S3支持原生USB，很多开发板通过USB的DTR/RTS信号自动控制GPIO0和EN实现"自动下载"——你点一下烧录按钮，电脑自动帮你按BOOT+RST。这需要加两个三极管搭一个自动下载电路（可以参考乐鑫官方DevKitC的原理图）。我们暂时用手动按键，简单可靠。

> **能不能不加BOOT键？** 如果你的项目只需要烧录一次固件之后就不改了，确实可以不加。但开发阶段需要频繁烧录，建议保留。后面生产时可以去掉。

## 第八步：画USB接口电路

### 为什么用USB Type-C？

USB接口在这个项目中有两个用途：
1. **烧录和调试**：通过USB给ESP32-S3烧写固件、打印调试信息
2. **充电**：USB的5V给TP4056充电芯片供电（这部分在电源管理电路里画）

### 为什么不需要USB转串口芯片？

很多老款ESP32开发板上有一颗CH340或CP2102芯片，用来把USB信号转换成串口(UART)信号，因为老款ESP32没有原生USB功能。

**ESP32-S3自带原生USB**（GPIO19=D-, GPIO20=D+），可以直接和电脑通信，不需要额外的转换芯片。这样省了一颗芯片、省了钱，也更简单。

> **另一种做法**：如果你想同时保留USB原生功能和传统串口调试，可以额外加一颗CH343（USB转串口芯片），通过UART0(TXD0/RXD0)和ESP32-S3通信。好处是USB原生接口可以用来做别的事（比如USB HID键盘模拟）。我们的项目不需要这么复杂。

### 电路设计

需要的元件：

| 元件 | 值 | 封装 | 数量 | 用途 |
|------|---|------|------|------|
| Type-C母座 J1 | 16P | - | 1 | USB接口 |
| 电阻 R3 | 5.1K | 0603 | 1 | CC1下拉 |
| 电阻 R4 | 5.1K | 0603 | 1 | CC2下拉 |

连接方式：

```text
              ┌──────────────────┐
              │   Type-C 母座     │
              │                  │
              │  VBUS ──────────────── 标记为 VUSB (5V，接到电源管理电路)
              │                  │
              │  D-  ───────────────── GPIO19 (ESP32-S3)
              │                  │
              │  D+  ───────────────── GPIO20 (ESP32-S3)
              │                  │         R3 (5.1K)
              │  CC1 ───────────────┤├──── GND
              │                  │         R4 (5.1K)
              │  CC2 ───────────────┤├──── GND
              │                  │
              │  GND ──────────────── GND
              │  外壳/屏蔽 ────────── GND
              │                  │
              └──────────────────┘
```

### 每个部分为什么这么接？

**VBUS (5V)：**

这是USB提供的5V电源。我们不直接用它给ESP32-S3供电（芯片要3.3V，5V会烧掉），而是引出一根线标记为`VUSB`，后面接到TP4056充电芯片和AMS1117稳压芯片，经过降压后再供电。

**D- 和 D+ (数据线)：**

USB的两根数据线，直接接到ESP32-S3的GPIO19和GPIO20。

为什么这两个引脚不能换？因为ESP32-S3内部的USB控制器硬件直连在GPIO19/20上，这是芯片设计时就固定的，没法通过软件改到别的引脚。

> **要不要加ESD保护？** 专业产品通常会在D+/D-线上加一颗ESD保护二极管（比如USBLC6-2SC6），防止静电损坏芯片。USB口经常插拔，很容易产生静电。加一颗才几毛钱，推荐加上。但为了简化教程，这里先不加。后面你可以自己补上。

**CC1 和 CC2 (配置通道)：**

这是Type-C特有的引脚，和老式Micro-USB最大的区别。

CC引脚的作用：当你把USB线插上去时，充电器/电脑通过检测CC引脚上的电阻来判断：
- **对面是什么设备？** 是手机（需要供电的）还是充电器（提供电源的）？
- **能提供多大电流？** 默认500mA还是更多？

我们接5.1K电阻到GND，这是在告诉对方："我是一个**用电设备(Sink)**，请给我供电"。

| CC下拉电阻值 | 含义 |
|-------------|------|
| 5.1K | 我是用电设备（这是我们要用的） |
| 无电阻(浮空) | 没有设备连接 |

> **如果不加CC电阻会怎样？** 很多Type-C充电器和电脑会认为没有设备连接，不会输出5V电源。你的板子插上USB线但不会有电。虽然有些老式充电器没有这个检测（直接给5V），但这是不规范的，不能指望。**5.1K电阻是必须的。**

> **其他选择**：如果你要做USB PD快充（比如9V/12V），就不能用简单的电阻了，需要用专门的PD协议芯片（如FUSB302）。我们的项目只需要5V供电，5.1K电阻就够了。

## 第九步：连线和网络标签

### 连线的两种方式

**方式一：直接画线**

点击工具栏的 **"导线"** 工具（快捷键 `W`），从一个引脚画到另一个引脚。

适合距离近的连接，比如电容和旁边的电源引脚。

**方式二：网络标签（Net Label）**

点击工具栏 **"网络标签"**（快捷键 `N`），给引脚贴一个名字。在另一个地方贴同名标签，两处就算连上了——不用画线。

适合距离远的连接。想象一下如果麦克风电路在原理图的左边，ESP32-S3在右边，中间拉好几根长线会很乱。用网络标签就干净多了。

> **为什么推荐用网络标签？**
> 1. 原理图更整洁、更易读
> 2. 如果以后改引脚分配，只需要改标签名，不用重新拉线
> 3. 在多页原理图中，不同页面之间只能用标签连接，不能拉线

### 预留给其他模块的网络标签

在ESP32-S3模组的对应GPIO引脚上，放置以下网络标签。它们现在只是"悬空"的标签，等后面画对应模块时，在那边也放同名标签就自动连上了。

| 标签名 | ESP32-S3引脚 | 将来连接的模块 | 信号说明 |
|--------|-------------|--------------|---------|
| `I2S_MIC_SCK` | IO14 | INMP441麦克风 | I2S时钟线 |
| `I2S_MIC_WS` | IO15 | INMP441麦克风 | I2S帧同步 |
| `I2S_MIC_SD` | IO16 | INMP441麦克风 | I2S数据输入 |
| `I2S_SPK_BCLK` | IO17 | MAX98357A功放 | I2S位时钟 |
| `I2S_SPK_LRC` | IO18 | MAX98357A功放 | I2S左右声道选择 |
| `I2S_SPK_DIN` | IO8 | MAX98357A功放 | I2S数据输出 |
| `SPI_SCK` | IO12 | ST7789屏幕 | SPI时钟线 |
| `SPI_MOSI` | IO11 | ST7789屏幕 | SPI数据线 |
| `LCD_DC` | IO10 | ST7789屏幕 | 数据/命令选择 |
| `LCD_CS` | IO9 | ST7789屏幕 | 片选（选中这块屏幕） |
| `LCD_RST` | IO46 | ST7789屏幕 | 屏幕复位 |

操作：逐个在ESP32-S3模组的对应引脚旁放置这些网络标签。

> **这些引脚能换吗？** 可以。ESP32-S3的大部分GPIO功能可以通过软件自由分配。比如你把I2S_MIC_SCK从GPIO14改到GPIO12，硬件上没问题，只需要在代码里也改一下就行。但有几个引脚**不能换**：GPIO19/20 (USB是硬件固定的)、GPIO0 (启动模式)。另外，GPIO26~32在有些模组上被内部Flash/PSRAM占用，具体看你买的模组型号的数据手册。

## 第十步：整理与检查

### 10.1 添加标注

双击每个元件，可以修改**位号**和**值**：

| 元件类型 | 位号规则 | 示例 |
|---------|---------|------|
| 电阻 | R1, R2, R3... | R1=10K (EN上拉) |
| 电容 | C1, C2, C3... | C1=100nF (去耦) |
| 模组 | U1 | U1=ESP32-S3-WROOM-1-N16R8 |
| 按键 | SW1, SW2 | SW1=RST, SW2=BOOT |
| 接口 | J1 | J1=USB Type-C |

**为什么要标位号？** 后面做PCB和焊接时，工厂需要知道"R1在哪里、是什么值"。位号就是每个元件的"身份证号"。

### 10.2 运行DRC检查

DRC = Design Rule Check（设计规则检查），相当于让软件帮你检查有没有画错。

1. 点击菜单 **"设计" → "设计规则检查"**
2. 查看结果：
   - **红色错误**（必须修）：引脚短路、电源接地冲突等
   - **黄色警告**（建议看看）：引脚未连接等
3. 常见问题及解决：
   - **"引脚未连接"**：检查是不是忘接了某个引脚。如果这个引脚确实不用，可以给它加一个"未连接"标记（放一个`X`符号在引脚上）
   - **"电源未连接"**：检查3V3和GND符号是否都正确连接
   - **"网络标签拼写不一致"**：比如一个写了`I2S_MIC_SCK`另一个写了`I2S_Mic_SCK`，大小写不同就不算同一个网络

### 10.3 最终布局建议

把原理图按功能区域整理，不要把元件随意散落：

```text
┌─────────────────────────────────────────────────┐
│  左上：USB Type-C接口    │  右上：ESP32-S3模组   │
│  + CC电阻               │  + 去耦电容            │
├─────────────────────────┤                        │
│  左下：EN复位电路        │  右侧引出：           │
│  + BOOT按键电路         │  各网络标签            │
│                         │  (I2S/SPI/GPIO等)      │
└─────────────────────────────────────────────────┘
```

> **为什么要整理布局？** 原理图的布局不影响实际电路功能（PCB才影响），但整齐的原理图方便你自己检查、方便别人看懂，以后排查问题也更快。

## 常见新手问题

**Q: 电阻电容的封装（0402/0603/0805）怎么选？**

这些数字是元件的物理尺寸代号：

| 封装 | 尺寸(mm) | 特点 |
|------|---------|------|
| 0402 | 1.0×0.5 | 非常小，手焊困难，适合量产贴片 |
| 0603 | 1.6×0.8 | 较小但手焊可行，推荐新手用 |
| 0805 | 2.0×1.25 | 较大，手焊轻松，占地稍多 |

建议：电阻选0603，小电容(100nF)选0603，大电容(10uF)选0805（因为大容量需要更大体积）。

**Q: 有些GPIO引脚我没用到，需要处理吗？**

不用。ESP32-S3内部有弱上拉/弱下拉可通过软件配置，没用到的GPIO浮空就行。不需要像某些老芯片那样给每个空闲引脚接上拉/下拉电阻。

但如果你想让设计更专业，可以把没用到的GPIO引出到排针或测试点上，方便以后扩展。

**Q: 为什么用模组不用裸芯片？**

| 对比 | 模组(WROOM-1) | 裸芯片(ESP32-S3) |
|------|--------------|-----------------|
| 内含 | 芯片+天线+Flash+晶振+匹配电路 | 只有芯片本身 |
| 你要设计 | 电源+信号线 | 电源+晶振+Flash+天线+RF匹配+... |
| 难度 | 新手友好 | 需要射频PCB设计经验 |
| 成本 | 模组单价贵几块钱 | 芯片便宜但要买更多外围元件 |
| 认证 | 模组已通过FCC/CE认证 | 裸芯片自己设计需要重新认证 |

结论：个人项目/小批量生产用模组，大批量(1万+)才考虑裸芯片方案。

**Q: 我能换GPIO引脚分配吗？**

可以，ESP32-S3的GPIO功能可以通过软件自由映射（叫做GPIO矩阵）。但以下引脚有特殊限制：

| 引脚 | 限制 |
|------|------|
| GPIO0 | 启动模式选择，不要做其他用途 |
| GPIO19, GPIO20 | USB D-/D+，硬件固定 |
| GPIO22~32 | N16R8模组中不引出，被Octal SPI PSRAM占用 |
| GPIO33~37 | N16R8模组中被Octal SPI Flash占用，GPIO35/36/37完全不可用 |

具体哪些引脚可用，以你买的模组数据手册为准。
