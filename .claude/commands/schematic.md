## 工作流程

当用户请求设计某个模块的原理图时，严格按照以下流程执行：

### 第一步：阅读资料

1. 读取 `/Users/mandy/Documents/GitHub/AI-bot/元件资料区/元件.md` 确认该模块使用的元件型号
2. 读取 `/Users/mandy/Documents/GitHub/AI-bot/元件TXT/` 下对应元件的数据手册文本
3. 读取 `/Users/mandy/Documents/GitHub/AI-bot/功能讨论区/task.md` 中的引脚分配参考
4. 读取 `/Users/mandy/Documents/GitHub/AI-bot/原理图设计/` 下已有的设计文档，确认已完成的模块和网络标签

### 第二步：分析与确认

从数据手册中提取以下关键信息：
- 供电电压范围和典型值
- 通信接口类型（I2S / SPI / I2C / UART / GPIO）
- 必须连接的引脚及其功能
- 推荐的外围电路（去耦电容、上拉/下拉电阻、滤波电路等）
- 数据手册中的典型应用电路（Application Circuit）

**关键规则：如果在以下任何情况下存在不确定，必须立刻用 AskUserQuestion 工具询问用户，不要自己猜测：**
- 数据手册中有多种应用电路，不确定选哪个
- 某个引脚有多种配置方式（如地址选择、增益设置、通道选择）
- 供电方案有多种选择（3.3V / 5V）
- 元件的具体封装型号不确定
- 引脚分配与已有设计存在冲突
- 需要额外的保护电路但不确定是否需要
- 任何可能影响最终PCB焊接和功能的决策

### 第三步：编写原理图教程

确认所有细节后，按以下格式输出教程文档：

1. **模块简介**：用一句大白话解释这个元件是干什么的
2. **你需要准备的元件清单**：列出所有需要放置的元件（含型号、封装、数量）
3. **分步操作指南**：
   - 每一步都写清楚：在嘉立创EDA里点哪里、搜什么、放在哪
   - 用 ASCII 示意图画出连接关系
   - **每个元件都必须解释三件事**：
     a. **为什么要加？** 这个元件起什么作用，不加会怎样
     b. **为什么用这个值？** 比如为什么是10K不是1K，为什么是100nF不是10nF，用通俗的语言解释选值的理由
     c. **还有什么其他接法？** 列出替代方案以及各自适用的场景（比如"如果你想要自动下载功能，可以改用两个三极管搭自动下载电路"）
   - 每个引脚的连接方式，都要解释这个引脚在芯片内部干什么、为什么要这样连
4. **与ESP32-S3的连接**：明确写出网络标签名和对应的GPIO
5. **检查清单**：列出画完后需要逐项确认的要点
6. **常见问题**：预判新手可能的疑惑并回答

### 第四步：保存文档

将教程保存到 `/Users/mandy/Documents/GitHub/AI-bot/原理图设计/` 目录下，文件名格式为 `{序号}_{模块名}.md`，序号按顺序递增（01已被ESP32-S3使用）。

## 写作风格要求

- 全部使用中文
- 不使用任何专业术语而不解释（如果必须用，紧跟括号解释）
- 每个操作步骤都要写"点击xxx"、"输入xxx"级别的具体指引
- 多用类比帮助理解（比如"去耦电容就像水管旁边的蓄水池"）
- ASCII电路示意图要清晰易读
- 宁可啰嗦也不要省略步骤

## $ARGUMENTS 处理

用户会传入想要设计的模块名称，如：
- `麦克风` 或 `INMP441` → 设计INMP441麦克风电路
- `功放` 或 `MAX98357A` → 设计MAX98357A音频功放电路
- `屏幕` 或 `ST7789` → 设计ST7789显示屏接口电路
- `电源` 或 `电源管理` → 设计TP4056 + AMS1117电源管理电路
- `传感器` 或 `MPU6050` 或 `QMI8658` → 设计六轴传感器电路
- `触摸` 或 `TTP223` → 设计触摸+按键电路

如果用户没有指定模块或输入不明确，用 AskUserQuestion 询问要设计哪个模块。
